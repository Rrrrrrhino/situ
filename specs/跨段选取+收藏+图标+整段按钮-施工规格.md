> 目标文件：几乎全在 `santu_app/index.html`（前端）。后端 `santu_app/app.py` 只可能动 0~1 处（`collect_keyword` 已现成，大概率不用动）。

## 背景：现有高亮/讲解架构（读懂再动手）

正文里每个词是 `<span class="w">` 或 `<mark class="vocab">`，句子是 `.sent`，段落块是 `p,h1,h2,h3,h4,li,blockquote`。

三种"带子"：
- **讲解带** `.phrase`：`wrapPhrase(range,text,sentence)` 生成，带 `data-pid / data-phrase / data-sentence / data-ex`（data-ex 存讲解 JSON）。点它 `reopenPhrase()` 重开讲解。
- **荧光笔带** `.hl`（kind=span）：`wrapHighlight(range,colorKey,'span')` 生成，带 `data-hl-id / data-c / data-kind`，内联 `style.background=var(--wX)`。
- **整段色卡** `.hl.para`（kind=para）：`_applyParaCard(block,colorKey,id)`，直接给 block 加 class + 更淡背景（`HL_PARA_BG`）。

书模式持久化：`_highlights[]` 数组镜像后端 `highlights` 字段，每条 `{id,ch,blockOrd,start,end,color,kind}`，`start/end` 是**块内字符偏移**。锚点换算函数：`_hlPointToBlockOffset` / `_hlComputeAnchor`（当前跨块返回 null）/ `_hlOffsetToPoint` / `_applyHighlightsForChapter`（换章重渲后按锚点重包）。文章模式的 `.phrase`/`.hl` 随 `readingSnapshot()` 的 innerHTML 快照天然持久，无需后端。

**当前限制（本批要打破）**：
1. `wrapPhrase` 开头 `if(!hostEl || !hostEl.closest('.sent')) return null` —— 选区不在单个 `.sent` 内（即跨句/跨段）就不生成讲解带，只临时讲解。
2. `mouseup` 处理有 `t.length>2000` 硬上限。
3. 高亮/讲解都是"单段单 span"，跨段落无解（`<span>` 套不住多个 `<p>`）。
4. `_hlComputeAnchor` 跨块返回 null。

## 任务 1：跨句 / 跨段都能"讲解 + 高亮"（核心，最重）

### 1a. 放宽篇幅上限
`mouseup` 监听里（约 L2435）：`t.length>2000` → 改成 `t.length>20000`。其余不动（`t.length<2` 保留）。

### 1b. 新增通用「按段多 span 包裹」helper

新增函数 `_wrapRangeAcrossBlocks(range, factory)`：
- 语义：把 range 按它跨过的段落块**逐段切开**，每段生成一个 span（由 `factory(block, isFirst, isLast)` 返回一个新建的空 span 元素），把该段内的选区片段包进去。所有 span 共享调用方在 factory 里写好的分组标识。返回 `{spans:[...], blocks:[...]}`（spans 与 blocks 同序，空段跳过）。
- 实现要点：
  1. 先 `liftRangeToWords(range)`（复用现有，避免切碎生词）。
  2. 找出 range 跨过的块列表：从 `range.startContainer` 与 `range.endContainer` 各自 `closest('p,h1,h2,h3,h4,li,blockquote')` 得首末块；用 `_bkBlocks()`（返回当前 DOM 里按序的块数组，已存在）取首末块之间（含）所有块，过滤出 `reading.contains(block)` 且与 range 真有交集的。
     - 判断"块与 range 有交集"：`range.intersectsNode(block)`（标准 API，可用）。
  3. 若最终只有 1 个块 → 退化为单段：直接对原 range 走单 span 包裹（用 factory 生成的 span + `surroundContents`，失败退 `extractContents/insertNode`），返回单元素数组。**单段路径必须与旧行为逐字节等价**（回归红线）。
  4. 多块：对每个块 `b` 构造"该块内的子 range"：
     - `sub=document.createRange(); sub.selectNodeContents(b);`
     - 若 `b` 是首块：`sub.setStart(range.startContainer, range.startOffset)`（try/catch，失败则保留块内容开头）。
     - 若 `b` 是末块：`sub.setEnd(range.endContainer, range.endOffset)`（try/catch，失败则保留块内容结尾）。
     - 中间块：整块内容。
     - 若 `sub.toString().trim()===''` → 跳过该块（不生成空 span）。
     - 用 `factory(b, i===0, i===last)` 生成 span，`try{ sub.surroundContents(span) }catch{ span.appendChild(sub.extractContents()); sub.insertNode(span) }`。收集成功的 span。
  5. 包裹会改 DOM/几何——**必须逐块从后往前包**（先包最后一个块，再往前），否则前面的包裹会让后面块的 range 失效。或者先把每个 sub-range 的锚点（node,offset）快照下来再逐个包——择一稳妥实现，自己在 preview 验证多块不串位。

> 注意：`_bkBlocks` 存在与否请先 `grep`；若命名不同（可能叫 `_bkBlocks()` 或类似），用实际的那个。找不到就用 `Array.from(reading.querySelectorAll('p,h1,h2,h3,h4,li,blockquote'))` 过滤 `reading.contains`。

### 1c. 讲解跨段：改造 `wrapPhrase` + `explainSelection`

- **打破单句闸**：`wrapPhrase` 里那句 `if(!hostEl || !hostEl.closest('.sent')) return null` —— 删掉"必须在 .sent 内"的要求，改成只要求"选区在 `reading` 内且至少落在一个段落块里"。跨句（同段内多句）现在应生成一条带子。
- **改造 `explainSelection`**（约 L2918）：
  - 现在逻辑：`wrapPhrase` 成功→带子+讲解；失败→临时讲解无带子。
  - 改为：用 `_wrapRangeAcrossBlocks(range, factory)`，factory 生成 `.phrase` span：**同一次选取生成的所有 span 共享同一个 `data-pid`**（一个新 pid），每个都带 `data-phrase=整段选中文本`、`data-sentence=_selSent`、`data-ex='{}'`。
  - 拿到 spans 数组后：`_curPhrase=spans[0]`（首个作代表，用于工具条定位与讲解写回），`showPhraseBar(spans[0])`，`await showPhrase(spans[0], true)`。
  - 若 spans 为空（极端：完全没包成）→ 退回旧的"临时讲解无带子"分支。
- **data-ex 要同步到同组所有 span**（关键，保证点任意一条都能重开、且文章快照持久）：新增 helper `_syncPhraseEx(ph)`：把 `ph.dataset.ex` 复制到所有 `reading.querySelectorAll('.phrase[data-pid="'+ph.dataset.pid+'"]')`。在两处写 data-ex 后调用它：
  - `showPhrase` 里 L2909 `ph.dataset.ex=JSON.stringify(r)` 之后 → `_syncPhraseEx(ph)`。
  - 追问写回 L1636 `ctx.phraseEl.dataset.ex=JSON.stringify(d)` 之后 → `_syncPhraseEx(ctx.phraseEl)`。
- **`removePhrase(ph)`**（约 L2485）：改成移除**同组所有** span（按 `data-pid`），不是只移除点中的那一个。逐个解开包裹（`while(firstChild) 提升到 parent；removeChild；parent.normalize()`）。`_curPhrase` 落在组内则清空。末尾 `scheduleSave()`。
- **点击分发**（约 L2327-2328）：`reopenPhrase(ph)` 保持——因为 data-ex 已同步到每个 span，点组内任意一条都能正确重开。

### 1d. 荧光笔高亮跨段：改造 `applyHighlight`

`applyHighlight(range, colorKey)`（约 L2555）：
- 保留"整段色卡"识别（单块且 `selTxt===blockTxt`）→ 走 `_applyParaCard`（不变）。
- 否则用 `_wrapRangeAcrossBlocks(range, factory)`，factory 生成 `.hl` span（kind=span），**同组所有 span 共享一个 `data-group` id**（新字段，一个新 group id），各自 `data-hl-id` 仍唯一、`data-c=colorKey`、`style.background=var(--wX)`。
- **书模式锚点**：对每个成功的 span，用它**自己那段的 sub-range**算锚点。做法：在 `_wrapRangeAcrossBlocks` 里，包裹**前**先对每个 sub-range 调 `_hlComputeAnchor(sub)`（现在 sub 一定在单块内，能算出 `{blockOrd,start,end}`），把结果连同 span 一起返回（factory 无法拿到，改为让 `_wrapRangeAcrossBlocks` 额外返回 `anchors:[...]` 与 spans 同序）。然后 `applyHighlight` 里书模式对每个 `{span,anchor}` push 一条 `_highlights` 记录：`{id:span.dataset.hlId, ch:bookIdx, blockOrd:anchor.blockOrd, start, end, color:colorKey, kind:'span', group}`——**新增 `group` 字段**。
- 末尾 `scheduleSave()`。

> 为最小化改动，`_wrapRangeAcrossBlocks` 返回结构建议：`{spans, anchors}`（anchors 与 spans 同序；单块退化时 anchors=[单锚点或 null]）。讲解 factory 用不到 anchors，忽略即可。

### 1e. group 化的改色 / 移除 / 重开（荧光笔）

- **`unwrapHl(el)`**（约 L2508）：若 el 有 `data-group` → 移除**同组所有** `.hl[data-group=X]`（span 逐个解包；同时从 `_highlights` 删掉所有 `group===X` 的记录）。无 group（旧单条记录）→ 保持原逐条行为（按 `data-hl-id`）。para 卡仍走 `unwrapParaCard`。
- **`recolorHighlight(el,colorKey)`**（约 L2522）：有 group → 改**同组所有** span 的 `data-c`/背景 + `_highlights` 里同 group 记录的 color。无 group → 原行为。
- **`reopenHl(el)` / `showHlBar`**：不变（工具条锚定点中的那条 span 即可，操作走 group）。
- **`_applyHighlightsForChapter`**（约 L3635）：重包时把 `rec.group`（若有）写回 `span.dataset.group`，保证换章重渲后同组仍能一起改色/移除。（para 分支不变。）

> 兼容：旧存档的 `_highlights` 记录没有 `group` 字段，按"无 group=独立单条"处理，不得报错、不得丢失。

- **新增 `paragraphCardFromSelection(range)`**：
  - 找出 range 跨过的所有段落块（同 `_wrapRangeAcrossBlocks` 的选块逻辑：首末块 + 中间、`reading.contains` 且 `range.intersectsNode`）。

  - 书模式：每块 push 一条 `_highlights` 记录 `{id, ch:bookIdx, blockOrd:_bkBlocks().indexOf(block), start:0, end:blockText.length, color:_lastParaColor, kind:'para'}`（照抄现有 `applyHighlight` 里 para 分支的 record 写法）。
