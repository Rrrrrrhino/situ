> **接续暗号：「做四土仅标记批α，先读 specs/仅标记-批α-生成管线-施工规格.md」**
> 前置阅读：`specs/仅标记-收获集-总体设计.md`（定案，不复议）＋ `~/Downloads/apply/style-codex/心法.md` §3（施工心法）。
> 本批**活到你为止**，不得再转包。核批（划痕层）已装机，数据源已就绪。

## 0. 你要做什么（一句话）

把用户读书时留下的标记（captures，已随存档落在 `LIBRARY/{doc_id}.json`）批量喂给
DeepSeek，生成「词块归位＋三档分级＋释义＋例句」的收获集条目，落盘供批β 渲染。

## 1. 输入（已存在，核批产出——别改它的写入方）

`~/…DATA_ROOT/library/{doc_id}.json` 里的 `captures` 数组，每条：
```json
{ "id":"cp…", "group":"cg…", "text":"fits and starts",
  "sentence":"Business came in fits and starts.",  // 可能为空串（无 .sent 包裹的段落）
  "ts":1751932800000,
  "ch":3,          // 书模式=章 idx；文章模式=null
  "blockOrd":2,    // 块序（含 h1/li 等所有块级，非「第N段」；文章模式含 doc-title）
  "start":120, "end":135 }   // 书模式锚点偏移；文章模式为 null
```
注意：同一次跨段划选产生多条同 `group` 的记录——**同 group 合并为一个条目**（text 拼接按 blockOrd 序）。

## 2. 输出（新文件，批β 消费）

`DATA_ROOT/sheaf/{doc_id}.json`（目录不存在则建）：
```json
{ "doc_id":"…", "updated_at":1751932800.0,
  "items":[{
    "cid":"cp…",                    // = capture id（group 合并时取首条）
    "raw":"toll",                   // 用户实际划的
    "chunk":"take a toll on",       // 归位后的词块（就是划的完整词块时与 raw 相同）
    "snapped":true,                 // chunk≠raw 时 true
    "tier":1,                       // 1|2|3，判据见 §4
    "def":"对……造成损耗、伤害（长期而渐进的）",
    "note":"主语常是压力、劳作、岁月；on 后接被磨损的人或物。",   // ≤40 字
    "sents":[{"en":"…原文句…","src":true},{"en":"例句2"},{"en":"例句3"}],  // 3–4 句，首句尽量=原文句
    "ch":3, "blockOrd":2, "ts":…,
    "status":"done"                 // done|error
  }],
  "pending":["cp…"]                // 已见 capture 但尚未生成的 id
}
```

## 3. 管线（Python，santu_app/app.py 内新增；不建新进程/线程池，一个后台线程够）

1. **触发**：`save_session` 每次写盘后，比对该 doc 的 captures 与 sheaf items/pending 差集；
   新增 ≥6 条、或距上次生成 >90 秒且有新增 → 起后台 daemon 线程跑一批（同 doc 有在跑则跳过，
   模块级 `_sheaf_running:set[str]` + lock）。**为什么边读边攒**：读完打开收获集应当已就绪，
   「读完即得」是本功能的魔法时刻。
2. **调用**：DeepSeek `deepseek-chat`（**绝不要 v4-pro**——这是例句/释义任务不是推理任务，
   v4-pro 慢且有 reasoning 吃 max_tokens 的坑，见 memory `reference_v4pro_reasoning_maxtokens_trap`）。
   每批 ≤10 条合一个请求；response_format json；max_tokens 3072；失败重试 1 次，仍失败该批
   items 标 `status:"error"` 留在 pending，下轮再试。API key 复用主窗现有 explainer 的 key 读法。
3. **prompt 骨架**（中文指令，见 §4 rubric；输出严格 JSON 数组）：每条给 `text/sentence`，
   要求返回 `chunk/tier/def/note/sents`。原文句放 sents[0]（`src:true`）；例句英文、地道、
   典型语境、12–20 词；**绝不要**造与原文句雷同的例句。
4. **删除同步**：captures 里已消失的 cid（用户取消了标记）→ 从 items/pending 里删掉。

## 4. 三档 rubric（写死进 prompt，配正反例；档义永久固定）

- **一档**（高频且地道）：母语者日常口语书面通吃、可迁移场景广。正例：take a toll on、stay put。
  反例：不能只因「常见词组成」就给一档——"three counties away" 不是词块。
- **二档**（常用）：常见但语域略窄或频率略低。正例：in fits and starts、play the long game。
- **三档**（低频/偏文学）：欣赏为主、日常输出可迁移性低。正例："a town that time itself
  seemed to have forgotten"。
- 判据 = 通用度 × 地道度 × 可迁移性，**相对通用英语**，与用户个人水平无关。
- **一致性红线**：同一表达任何时候重跑必须同档——prompt 里给满正反例就是为了这个；
  抽验方式见 §6。

## 5. 范围围栏

- 只动 `santu_app/app.py`（新增方法/线程）＋新建 `DATA_ROOT/sheaf/` 目录；**不碰** index.html
  （批β 的事）、**不碰** `mobile/`、**不碰**核批写好的 captures 读写逻辑、**不删**任何现有代码。
- 新增桥方法：`get_sheaf({doc_id})` → 返回上述 JSON（批β 用）；`regen_sheaf({doc_id, cids?})`
  → 强制重跑（全部或指定条目）。
- 写盘一律原子写（临时文件+rename，学 index.json 之前踩过的并发半截坑）。

## 6. 验收（可执行）

1. 单测（mock LLM）：6 条 captures（含一组跨段同 group、一条已取消的）→ items 合并正确、
   删除同步正确、pending 收敛为空。贴测试输出。
2. 真跑：拿用户任一篇有 ≥8 条标记的存档（没有就自己在 App 里标几条），跑一轮，贴
   `sheaf/{doc_id}.json` 全文——检查：词块归位是否合理、例句是否地道、tier 是否符合 §4。
3. **一致性抽验**：同一批 captures 重跑一遍，比对两次 tier——不一致条目 >1/10 = 返工
   （先收紧 rubric prompt 再重测）。
4. `python3 -m py_compile app.py` 过；**装机不归你**（批β 一起装）。
5. 汇报=贴证据（测试输出/生成 JSON/两次 tier 对比表），不贴结论。
