# 仅标记 + 收获集 —— 总体设计（顶层定案）

> 状态：设计**全部定案**（2026-07-08，用户已拍板捕获反馈=方案 D，收获集=全变色方案）。
> 交互样机：`previews/仅标记-捕获反馈-mockup.html`（桌面副本在 `~/Desktop/预览/四土/`）——
> 施工时视觉/动效**逐字照抄样机 CSS**，它就是完工样例。

## 0. 一句话

阅读时只划不讲（心流不断、划痕即收藏），读完打开「收获集」——AI 已在后台把所有标记
批量做成「词块 + 例句 + 释义 + 三档分级」，档色此刻才揭晓；收获集可对照原文、可按档位
排序、可勾选喂给复述练习。

**为什么**：用户读纸质书的真实体验——逐词查断节奏，但「标记到一个可用表达」本身愉悦；
痛点只在标完还得手动逐个查。本功能把「查」整段外包给 AI，把「标」的愉悦保住。

## 1. 已拍板决策（每条附为什么，施工不得复议）

1. **模式开关**：阅读页工具胶囊加「仅标记」开关（与现有 首页/书签/复述 并列）。
   默认态暂维持普通讲解模式；是否让仅标记成默认，用户使用后再定。
2. **捕获反馈 = 方案 D（用户拍板）**：墨迹显影（划痕沿划向 220ms 迹显、落笔深色沉定
   `rgba(180,138,62,.46)`→素色 `rgba(190,150,80,.28)`，同 `--marker` 荧光调）
   ＋页脚拾数（「本次拾得 · N」muted 小字，pop 0.2s）＋耳语级石墨纸声（50ms 带通噪声
   1400–1900Hz、gain .07 起指数衰减；**设置里必须可关**）。
   **绝不在阅读中显示档位颜色**。为什么：即时判色=阅读中被打分，会诱导「为绿色而标」，
   污染标记本身的愉悦（风格圣经「事前胡萝卜」禁令）；且后揭晓=阅读时零 AI 调用。
3. **揭晓 = 显影，不做仪式页**：打开收获集内容即刻可读（零门槛、零点击、零等待）。
   显影形态（配合全变色方案）：词块标题先以正文墨色 `--fg` 上屏，档色以 0.55s
   `transition:color` 「洇」进字里，档签同步淡入，条目间约 90ms 错峰。
   为什么：用户明言「勤奋标完渴望立刻看讲解」，任何门都是摩擦。
   **绝不要**：全屏揭晓动画、逐条翻牌、需要点击的「开启」按钮。
4. **分档 3 档，锚死 rubric，档义永久固定；呈现走「整区变色·墨化」（样机③页方案「甲改」）**：
   - 档色（标题+档签）：一档「高频且地道」绿 `#5f7a58`／二档「常用」琥珀 `#b0894a`／
     三档「低频·偏文学」暖灰 `#8d8271`。
   - **整条讲解区（中英文例句+释义+用法注）全部着「档色墨」**——即档色的深一阶正文级：
     一档 `#4f6849`、二档 `#8a683a`、三档 `#6b6357`。词块标题用原档色（衬线 600），
     **无左侧色条**（全变色本身已足够强调）；档签=淡底印刷章（档色字＋档色 13–15%
     透明衬底），**绝不要**白字实心块。
   - 为什么分两阶：用户要「整个讲解区都变色，视觉吸引力才足够」；但原档色直接铺正文
     有两个实测失败点（样机③页可复看）——二档琥珀长例句发飘、三档暖灰全文像禁用文字。
     「档色墨」保住可读性同时保住整区色感。若用户看样机后改选甲（原档色铺满）或
     乙（半染：仅标题+右列着色），只改这三行正文色定义即可。
   - **例句内的核心词块 = 仅加粗（font-weight:700，color:inherit），无任何高亮底**
     （用户拍板，替代早先的素色荧光底方案）。小注（原文·第 N 段）保持 muted 灰。
   - 分档**相对通用英语**（常用度×地道度×可迁移性），**不结合个人词汇档案**——
     用户拍板：标了即代表不熟，个人化是画蛇添足。
   - prompt 里写死三档判据并各配 2 个正反例。为什么：AI 打档一旦时绿时橙，
     用户两次后就不再信颜色，整套分级作废。
5. **粗划精修**：用户随手划（哪怕只划中一个词），AI 生成时自动归位到所在最优词块，
   条目里显示「你划的是『X』—— 已归位到它所在的完整词块」。
   为什么：把标记精度要求降到纸质荧光笔水平，是全案摩擦力的大头；
   同时「鼓励词块」不再需要教育用户。
6. **讲解规格（用户点名的版式）**：左列例句 3–5 句竖排紧凑（首句=原文句，带
   「原文·第 N 段/章」小标），右列释义单独一列（释义 1 行 + 用法注 1 行，--han 字体）。
   **不做**「适用/不适用场景」栏——复述时 AI 反馈已覆盖，加了内容过载。
   为什么：终极目的是使用不是阅读讲解，讲解投入「适当」即可（用户原话）。
7. **收获集形态**：有边界的「册」，非必须落盘文件。
   - 视图：按章节序（对照用）⇄ 按档位（学习用），一个 seg 切换。
   - 范围：默认=本次阅读会话所及；可任选章节/段落范围重新生成。
   - 对照阅读：桌面上收获集可作右侧栏与正文并列（同 .panel 372px 语法）。
   - 导出：自包含 HTML 一键存（沿用项目惯例，天然可分享，为远期读书会埋线）。
   - **随用随调**：阅读页入口一步直达本书收获集；历史册进书架侧「册架」（与批三
     剪报盒同族，做的时候合并考虑）。
8. **复述衔接**：收获集条目可勾选 → 走既有复述题签卡管线（`retell_targets` 的
   chunks 注入口已在批⁶ 建成），默认按档位优先排目标表达。
9. **读书会（远期，搁置但记录）**：用户否决「导出文件互传合册」（摩擦太高，
   "不是 AI 时代该发生的事"）；要求未来在应用内实现多人交互（顺带提升使用率）。
   合册呈现用户**基本同意但要求深挖**：倾向只显「共鸣计数」（N 人标了这条），
   昵称+个人色是否显示未定；**绝不做任何人际比较**（谁标得多/少）。动工前必须再议。

## 2. 技术地基盘点（已存在，直接复用，不重造）

- **划选基建全套已建成**（批二 色块高亮）：`selbar` 浮条、`.hl` 内联 span 高亮
  （box-decoration-break:clone）、持久化走 `readingSnapshot()/doSave()` 内联落盘。
  仅标记的划痕 = 同族内联 span（类名建议 `.capt`，与 `.hl` 并存不混用）。
- **词讲解管线**（讲/prewarm）：例句+释义生成可参考其 prompt 与失败重试写法。
- **复述管线**（批⁶）：`retell_targets`/复述题签卡/chunkFeedback，衔接点现成。
- **LLM**：生成走 `deepseek-chat`（v4-flash）——例句/释义/分档是非推理任务，快且省；
  **绝不要**用 v4-pro（慢且 maxTokens 坑，见 memory）。每批 8–12 条合一次调用。
- **后台节奏**：边读边攒，每攒满 8 条或阅读 idle 60s 触发一批；读完打开收获集时
  未完成的条目以骨架呈现、`.ink-in` 渐次补齐（主窗已有同款骨架语法）。

## 3. 数据模型草案（批次施工前由 Fable 5 定稿）

- 划痕（随文持久，同 .hl 机制）：`<span class="capt" data-cid="…">`。
- 捕获记录 store（IndexedDB/服务端 JSON 与现有笔记同盘）：
  `{cid, srcId(书/文), chapter, rawText, ts}`
- 收获集条目（生成物）：
  `{cid, chunk, snapped(bool), tier(1|2|3), def, note, sents[{en, isSource, srcRef}], status}`
- 册：`{sheafId, srcId, range, items[], createdAt}`——导出/对照/复述都从册取。

## 4. 批次划分（用户指令：Fable 5 统筹+做最要害，其余给 Opus）

| 批 | 谁 | 内容 | 前置 |
|---|---|---|---|
| 核 | **Fable 5 亲做** | 仅标记模式开关 + 划痕层（词界吸附/方案 D 反馈/持久化/取消）+ 数据模型定稿 | 无（设计已全定） |
| α | Opus（另窗，规格另立） | 后台生成管线：攒批→flash 调用→分档+词块归位→落库→失败重试 | 核 的 store 定稿 |
| β | Opus（另窗，规格另立） | 收获集视图（全变色/双排序/显影/对照侧栏/导出 HTML/入口+册架） | α；样机=完工样例 |
| γ | Opus（可并入 β） | 复述衔接（勾选→retell_targets） | β |

**施工顺序=严格串行：核 → α → β(+γ)**，不并行。为什么：①核的划痕 store schema 是
α/β 的地基，先定谁都别动；②α 与 β 都要动主窗 `index.html` 同一文件，两个 Opus 窗口
并行改必然打架（本项目无分支工作流）；③串行让 Fable 5 每批验收后再放行下一批，
返工半径最小。每批之间 Fable 5 验收（审证据+亲跑最要害 1–2 条）。

- 批α/β 规格由 Fable 5 照心法 §2 六条另立（本文件不是施工规格）。
- **范围围栏（各批通用）**：不碰 `mobile/` 手机分叉；不动 selbar 现有「讲解」按钮
  行为与七色高亮语义；不删任何现有死代码；正文渲染管线只加不改。

## 5. 验收骨架（各批规格里细化成可执行条目）

- 核：仅标记模式下划/点/取消 30 次无一错位；关模式后讲解路径零回归；重开书划痕还原。
- α：50 条真实标记一次跑通，档位分布合理且**同批重跑档位一致**；单批调用≤12 条。
- β：打开收获集 time-to-content < 300ms（内容先于档色）；导出 HTML 双击自包含可看；
  截图过眼与样机成套。
- 全链路：真书一章实读——标 15 条→读完开册→勾 3 条复述，全程不碰键盘配置。
