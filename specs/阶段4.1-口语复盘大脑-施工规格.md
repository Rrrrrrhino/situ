# 四土 · 阶段 4.1：口语复盘大脑（施工规格）

> 总蓝图见 `specs/手机原生app-施工规格.md`。新功能「口语复盘」全貌：录我说的英文 → AI 复盘我
> 输出的内容(哪好/哪错/换种更自然) → 自动进错题本 → 错题本拿来训练(口播/对话/写作)。
> **分步**：4.1 复盘大脑(粘贴/打字→复盘→错题本，纯 JS、浏览器可验、可同步电脑) → 4.2 错题本+训练
> → 4.3 接麦克风 + 火山 ASR。**本规格只做 4.1。**
> 主会话(Opus)已设计 prompt(净新、产品价值核心)+数据模型+接口+UI；builder 照此施工，**prompt 逐字
> 照搬一字不改**，汇报贴证据(尤其自测原始 JSON)不贴结论。

## 0. 这一步交付什么
在现成手机 UI(`santu_app/mobile/`)里**新增一个「复盘」视图**：贴/打一段你自己产出的英文 → 调
LLM 复盘 → 渲染成样式图那样的三栏(用得好/要改/换种更自然)+原文高亮 → 错误和"更自然"两类**自动
存进新的错题本(IndexedDB)** → 一个基础的错题本列表能看到、能标掌握/重点/删。**不做录音(4.3)、
不做训练出题(4.2)**——"练这几条"按钮先占位(路由到错题本或 toast「4.2 再做」)。

样式图(主会话已给用户看过并认可)的结构与配色就是渲染目标：暖纸底 `#faf8f4`、UI 深蓝 `#16335c`、
原文里**错误用赭橙高亮**、**不自然用暖金高亮**；三栏图标+标题色：用得好=橄榄绿、要改=赭橙、
换种更自然=暖金。这套是低饱和档案高亮色，和暖纸天然成套，别用鲜亮塑料色。

## 1. 复盘 Prompt（逐字照搬，这是产品价值，一字不改）
端口进 `js/core/llm.js`，命名 `REVIEW_SYSTEM` / `REVIEW_TEMPLATE`，风格与现有 7 条 prompt 一致。

### REVIEW_SYSTEM
```
你是一位资深的英语口语外教，专门帮"中文母语"的学习者复盘他们自己说出口或写出来的英文。学习者会给你一段他本人产出的英文（可能是把口播录音转写来的，会有口语的不完整、重复、口头语，这些口语特征不要苛责）。你的任务：客观、具体地复盘，帮他下次说得更准、更地道。

你只能输出一个 JSON 对象，结构如下（不要输出任何额外文字，不要 markdown 代码块）：
{
  "strengths": ["具体夸某个词 / 某个表达用得好，1-3 条；没有就给空数组"],
  "errors": [
    {
      "original": "学习者原文里出错的最小连续片段，必须逐字照抄原文",
      "correction": "改正后的说法",
      "type": "grammar | wordchoice | collocation 三选一",
      "why": "一句中文，说清为什么错或背后的规则"
    }
  ],
  "naturalness": [
    {
      "original": "学习者原文里的连续片段，必须逐字照抄原文",
      "better": "母语者更自然的说法",
      "why": "一句中文，说清语感差别 / 为什么这样更自然"
    }
  ]
}

判定与口吻要求：
- errors 只放"真正的错误"：语法错、用错词（词不达意）、搭配错。naturalness 放"没有语法错、但母语者一般不这么说"的表达。两者都没有就如实给空数组。
- original 字段必须是学习者原文里"逐字、连续"出现的片段（系统要用它在原文里高亮定位）：原样照抄，不要改写、不要翻译、不要加引号或省略号。
- 中等严格、抓大放小：抓真错和"有意义的地道升级"即可。口语填充词（you know / like）、轻微重复、可接受的口语化表达，不要挑。
- 绝不为了凑数硬造问题。如果这段英文本来就不错，errors / naturalness 就少给或给空数组。
- strengths 要具体（夸到具体的词或表达），不要泛泛地说"整体不错"。
- 所有 why、解释一律用简短中文，一句话讲清即可。
```

### REVIEW_TEMPLATE（user 消息；`{context_block}` 见下）
```
【学习者产出的英文】
{transcript}
{context_block}
请严格按系统要求复盘，只输出那个 JSON 对象。
```
- 有背景时 `{context_block}` = `\n\n【背景 / 对方说了什么 / 话题】\n{context}`；无背景则为空串。

## 2. llm.js：新增 `reviewSpeech(args)`
- 签名 `reviewSpeech({ text, context }) -> { ok, strengths, errors, naturalness, model, error? }`。
- **结构镜像现有 `explainWord`**：同样读 `store.settingsGet` 拿 provider/model/key；同样 JSON
  mode；同样 `_extractJson` 容错；**同样的"空答重试"**（v4-pro 是推理模型，先吃 reasoning token，
  `max_tokens` 给小了正文为空——见全局 memory `reference_v4pro_reasoning_maxtokens_trap`）。
- `max_tokens` 给足（≥ 2048，复盘条目可能多）；空/不合法 JSON 重试一次。
- 校验返回：三个字段都强制成数组；`errors[i]` 缺字段时丢弃该条而非整体失败。
- 缺 key 时返回 `{ ok:false, error:'未配置 API Key，去设置里填' }`（与现有风格一致）。

## 3. store.js：两张新表
照 `store.js` 现有 IndexedDB 表/helper 模式新增（**bump DB version**，加 onupgradeneeded 建表）：
- `reviews`（复盘记录，可回看历史）：
  ```ts
  { id, createdAt:number, transcript:string, context?:string,
    source:"paste"|"type"|"mic", model:string,
    result:{ strengths:string[], errors:ErrItem[], naturalness:NatItem[] },
    mistakeIds:string[] }
  ```
- `mistakes`（错题本）：
  ```ts
  { id, original:string, correction:string,
    type:"grammar"|"wordchoice"|"collocation"|"naturalness",
    severity:"error"|"naturalness", why:string,
    reviewId:string, addedAt:number, lastSeen:number,
    reviewCount:number, mastered:boolean, star:boolean }
  ```
- id 用现有生成方式（如 12 位 hex）。helper：`reviewsPut/reviewsList`、`mistakesPut/mistakesList/
  mistakesGet/mistakesDelete`，与现有 store helper 命名风格一致。

## 4. localapi.js：新增方法（I/O 形状，UI 照此调）
```ts
review_speech({ text, context?, source? }) ->
  { ok, reviewId, strengths:string[],
    errors:{ original, correction, type, why, mistakeId }[],
    naturalness:{ original, better, why, mistakeId }[],
    model, error? }
// 内部：调 llm.reviewSpeech → 写 reviews 一条 → 每个 error/naturalness 建一条 mistakes
//   (severity 对应 error/naturalness；naturalness 的 correction 存 better) → 回带 mistakeId 的结果
list_mistakes({ type?, view?, q? }) -> Mistake[]
//   view ∈ "all"|"unmastered"|"mastered"|"star"（默认 unmastered，照生词本习惯）
//   type 过滤 grammar/wordchoice/collocation/naturalness；q 关键词搜 original/correction/why
set_mistake_mastered({ id, mastered? }) -> {ok}
set_mistake_star({ id, star? }) -> {ok}
delete_mistake({ id }) -> {ok}
list_reviews() -> { id, createdAt, snippet, errorCount, naturalnessCount }[]   // 倒序，snippet=transcript 前若干字
```
`api()` 已是 `LocalApi[method]` 分发，新增方法挂上即可。

## 5. UI：新增「复盘」视图（app.js + style.css，复用现有模式）
- **入口**：在现有导航(阅读/生词本/设置那组)加一个「复盘」。复用现有视图切换、sheet、列表/筛选机制，
  别另造框架。
- **输入屏**：一个大 `textarea`（占位「把你刚说的英文贴进来，或直接打」）+ 一个可选 `背景/话题`
  小输入框 + 「复盘」按钮。**麦克风入口本阶段不做**（4.3 再加），别放假按钮误导。
- **结果屏**（**严格照样式图**）：
  1. 顶部 meta（来源/词数/时间，时间可省）。
  2. **原文卡**：把 `transcript` 整段显示；用每个 `errors[].original` / `naturalness[].original`
     在原文里做**子串匹配高亮**（error=赭橙底+赭橙下划线、naturalness=暖金底+暖金下划线，色值照
     样式图）。匹配不到就不高亮该条（仍在下面列出），别报错、别崩。
  3. **用得好**（橄榄绿）：列 strengths。
  4. **要改**（赭橙）：每条卡片 `原话(删除线) → correction(深蓝)`，下面一句中文 why，一个 type 胶囊
     （语法/词不达意/搭配），一个「已进错题本」标记。
  5. **换种更自然**（暖金）：每条 `原话 → better(深蓝)` + 中文 why + 「不自然」胶囊。
  6. **底栏**：「N 条已存进错题本」+ 按钮「错题本」(去列表) +「练这几条」(本阶段占位：toast 或灰态)。
  - 英文用衬线(项目的 Bitter / Georgia 兜底)，中文讲解用霞鹜文楷(`'LXGW WenKai'`，本机已装)。
- **错题本列表屏**：复用 `vocab.js` 的列表/筛选/展开/星标/掌握/删除/搜索那套，数据换成
  `list_mistakes`。tab：未掌握/重点/已掌握/全部；按 type 筛；点条目展开看 why；星标/标掌握/删。
- **配色**：只新增三类高亮/标题色(赭橙/暖金/橄榄绿)，必须与暖纸和谐(低饱和档案色，照样式图 hex)，
  深蓝 `#16335c` 仍是 UI 强调色。

## 6. 分步施工 + 验收（每步贴证据）
- **Step 1 · 大脑先通（关键，先报这步）**：store 两表 + `llm.reviewSpeech`(逐字 prompt) +
  `localapi.review_speech` + 错题本 CRUD。写 `_devtest/test_review.mjs`(仿现有 `_devtest/test_llm.mjs`)，
  对**真实 DeepSeek**(默认配置；若有 v4-pro 用 v4-pro)跑两段并**把原始 JSON 贴回**：
  - (a) 有错的一段（可用样式图那段：`Yesterday I go to the park with my friend. The weather was
    very good so we very enjoy it. I want to improve my English level, so speaking more is the
    best way. Anyway, it is a meaningful day.`）→ 看是否抓到 go/very enjoy/it is 的时态与 very+动词
    错、English level 的中式表达，且 `original` 逐字可定位。
  - (b) 一段本来就不错的英文 → 看是否**不硬造**错误（errors/naturalness 该空就空）。
  贴：两段的原始 JSON 全文 + `node --check` 通过。
- **Step 2 · UI**：复盘输入/结果/错题本列表三屏（照 §5）。`node --check app.js vocab.js`(若动) +
  `npm run build`(core.bundle.js 重打) 成功。
- **Step 3 · preview 闭环**：本地起服务 → 开「复盘」→ 贴 (a) 那段 → 看三栏渲染 + 原文高亮 →
  确认错题本表新增 3 条、列表能看到 → 标一条掌握/重点/删各试一次 → console 零报错。
  贴：结果屏截图(视觉验收点) + 错题本列表截图 + console 日志。
- **Step 4 · 报告 + 不破坏**：确认没碰阅读流程/桌面 app.py；更新 `santu_app/mobile/CHANGELOG.md`
  阶段4.1 段 + 同步中央改动日志库。一句话回报：大脑通否、UI 通否、prompt 自测质量、未决问题。

## 7. 红线 / 别做
- **prompt 逐字照搬 §1**，不许"优化"措辞（它是判断重产物，主会话验收会比对）。
- **外科手术**：只新增「复盘」相关代码；不改阅读/讲解/生词本既有逻辑(列表 UI 可复用但别改坏)；
  不碰桌面 `app.py`/`index.html`；只删因本次改动失效的东西，原有死代码别动。
- **不做**：录音/ASR(4.3)、训练出题与批改(4.2)、深色模式、图标美化。"练这几条"占位即可。
- v4-pro 切记 `max_tokens` 给足 + 空答重试（否则复盘正文被 reasoning 吃空）。
- 错题本是个人学习数据，全本地 IndexedDB，不上传、不进代码。

## 8. 同步电脑版（顺带，不额外加工）
复盘大脑是纯 JS(文本+LLM+存储)，`santu_app/server.py` 已能把整个 `mobile/` 伺服到桌面浏览器，
所以**电脑上用同一个 `mobile/` 就能跑同一个复盘功能**——本阶段只要"别破坏 server.py 伺服路径"，
不用为桌面单独写代码。真正的跨端数据同步沿用现有 vocab_export/import 思路，留后续按需。
