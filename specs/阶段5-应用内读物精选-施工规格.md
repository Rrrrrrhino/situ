## 目标
应用内加一个「读物精选」入口：选媒体来源（CNA / The Conversation）→ 拉它的
RSS 看**近期文章列表** → 点某篇 → 走**现有**网址阅读管线进入阅读。用户不再手动粘贴链接。

## 复用现状（已查证，别重造）
- 抓网页已通：`js/core/http.js` 的 `httpGet(url, headers)`（Flutter 里走原生 GET 桥、
  浏览器里 fetch 回落）。**RSS 抓取直接用它**。
- 点一篇文章进入阅读 = 调用 **app.js 已有的全局 `addSource(url)`**
  （它内部 `processStart(() => api('process',{source,level}))` → mountDoc）。**别自己写阅读逻辑**。
- 菜单是 `data-nav` 驱动：index.html `#drawer nav button[data-nav]` →
  app.js `go(nav)` 分发。口语复盘(review.js)就是「独立 script + `window.renderReview` +
  drawer 加一项 + go() 加一分支」的范式，**本功能照抄这个范式**。
- 视觉复用现成类：`cont-card`（继续阅读卡）、`lib-item`（书架行）、`sec-h`（小节标题）、
  `loading-line`+`.spin`（载入）、`errBox(msg)`、`el(tag,cls,html)`、`esc()`、`timeAgo(ts)`、
  `toast()`、`showFull(html)`、`closeDrawer()`、`setTopTitle()`、`hideBookChrome()`
  —— 这些都是 app.js 里的全局，discover.js 在 app.js 之后加载即可直接用。

## 施工清单（5 个文件，全在 santu_app/mobile/）

### 1) `js/core/localapi.js` —— 新增 `fetch_feed`
- 加 `async function fetch_feed({ url } = {})`：
  - `httpGet(url)`（已 import）取 XML 文本；status≥400 或空 → 返回 `{error:"…"}`（分级文案）。
  - 用 `new DOMParser().parseFromString(text, "application/xml")` 解析。
  - **同时兼容 RSS 与 Atom**：
    - RSS：条目 = `item`；title=`item>title`；link=`item>link` 的**文本**；
      date=`item>pubDate`；summary=`item>description`。
    - Atom：条目 = `entry`；title=`entry>title`；link=`entry>link[rel=alternate]` 的
      **href 属性**（没有 rel 就取第一个 `link` 的 href）；date=`entry>published`||`updated`；
      summary=`entry>summary`||`entry>content`。
  - summary 要**去 HTML 标签**（`.replace(/<[^>]+>/g,' ')`）+ 解码实体 + 截断 ~100 字。
  - title 也要解码 HTML 实体（如 `&#039;` `&amp;`）——用一个 textarea/DOMParser 解码小工具。
  - 最多返回前 **30** 条；过滤掉没有 link 的条目。
  - 返回 `{ ok:true, feed_title:<channel/feed title>, items:[{title, link, date, summary}] }`。
- 在本文件的 **api 分发表**里注册 `fetch_feed`（跟 `process`/`process_file` 同一张表，
  照现有写法加一行）。
- 改完 **`cd santu_app/mobile && npm run build`** 重出 `js/core.bundle.js`（fetch_feed 在 bundle 里）。

### 2) `discover.js` —— 新文件（放 santu_app/mobile/ 根，跟 review.js 平级）
定义 `window.renderDiscover = async function(){…}`。两级视图，都渲染进 `#app`：

- **OUTLETS 注册表**（写死在文件顶部，已全部验证可用）：
  ```js
  const OUTLETS = [
    { id:'cna', name:'CNA', desc:'Channel NewsAsia · 亚洲视角英文新闻',
      feeds:[ {name:'最新', url:'https://www.channelnewsasia.com/rssfeeds/8395986'} ] },
    { id:'tc', name:'The Conversation', desc:'学者撰稿 · 深度解读',
      feeds:[ {name:'全球', url:'https://theconversation.com/global/articles.atom'},
              {name:'美国', url:'https://theconversation.com/us/articles.atom'},
              {name:'英国', url:'https://theconversation.com/uk/articles.atom'} ] },
  ];
  ```
- **第一级（来源列表）**：`setTopTitle('读物精选'); hideBookChrome();`
  一个 `sec-h`「选择来源」+ 每个 outlet 一张卡（复用 `cont-card` 观感）：粗体 name +
  小字 desc + 右侧 `›`。点卡 → 进第二级。
- **第二级（文章列表）**：
  - 顶部一行：`‹ 读物精选`（返回一级）+ 当前 outlet 名。
  - 若该 outlet 有多个 feed：一排小胶囊切换（复用现有胶囊/按钮小样式；选中态用深蓝 `#16335c`），
    默认第一个。单 feed 则不显示胶囊。
  - `loading-line` 载入中 → `api('fetch_feed',{url})`。
  - 成功：每篇文章一行（复用 `lib-item` 观感）：标题（两行截断）+ 小字日期（能解析成
    `timeAgo` 就 timeAgo，否则显示原 date 串）+ 可选一行 summary 灰字。点行 → **`addSource(item.link)`**。
  - 失败/空：`errBox(...)` + 一个「重试」按钮。
- 交互：点文章后由 addSource 接管（它会 closeSheet/showFull 进阅读），discover 不用自己清理。

- 在 `<script src="app.js"></script>` **之后**、review.js 附近加：
