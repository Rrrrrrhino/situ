# 口语复盘 · 转写热词/上下文注入 — 施工规格

> 施工方：**Opus 主会话**（用户指定这类活不占 fable5 额度）。接续暗号：
> **「继续弄四土口语复盘的热词注入，先读 specs/口语复盘-热词注入-施工规格.md」**
> 动手前先读项目根 `HANDOFF.md` 顶部两批（提速批 + 历史批），别撞车。

## 背景 / 目标

用户口语录音里的专有名词被火山 ASR 误识：说 "Fable 5" 被转成 "fibonacci"、
"casual talk" 被转成 "camera talk"。火山大模型录音文件识别支持**热词/上下文**参数
（`corpus` 字段，含 `boosting_table_name` / `context` 等，见
https://www.volcengine.com/docs/6561/155739 与大模型录音文件识别 API 文档）。
目标：让用户在设置里维护一份「常用专有名词」清单，转写时注入，专治这类误识。

## 改动点（外科手术式，只碰以下四处）

1. **设置界面** `santu_app/mobile/settings.js`（「语音转写（火山，可选）」区，L76-83 附近）：
   加一个多行输入 `textarea#setVolcHotwords`，label「转写热词（常用专有名词，每行一个）」，
   placeholder 给例子（Fable 5 / Sesame / DeepSeek / Anthropic…）。存 `volc_hotwords`
   （字符串原样存，解析在用的时候做）。保存走现有 `save_settings` 一起带上。
   注意 `get_settings`/`save_settings` 在 `js/core/localapi.js`，要把新 key 接进去（照
   volc_appid 的现有做法抄）。

2. **前端管道** `santu_app/mobile/js/core/localapi.js` `process_dualtrack`：
   `settingsGet("volc_hotwords")` → 解析成去重非空词数组（按换行/逗号分隔，cap 100 个）→
   随 `httpPost("/api/dualtrack_transcribe", {}, { dir, appid, token, hotwords })` 传给后端。

3. **后端注入** `santu_app/server.py`：
   - `_dualtrack_transcribe` 接 `hotwords`（list[str]），透传给 `_volc_transcribe_track` →
     `_transcribe_slice` → 极速版 `_transcribe_slice_flash` 和标准版 `_transcribe_one_slice`
     两条路的请求体都要注入。
   - **注入格式以官方文档为准，动手前先查**（文档页是 JS 渲染的，直接浏览器打开或搜
     "大模型录音文件识别 corpus context 热词"）：大概率是
     `request["corpus"] = {"context": "<热词逐词拼接或 JSON>"}`；`boosting_table_name`
     是控制台预建热词表方案，**不用**（要用户去控制台维护，违背「设置里一格搞定」）。
   - **容错铁律**：带 corpus 的请求若报参数错（4xx / 非 20000000），**去掉 corpus 重试一次**
     再走原有回退链——热词是增强，绝不能因为它把转写整个搞挂。极速版是否支持 corpus
     未验证过，必须真机验证（见下）。

4. **凭证/热词不落盘不打日志**（沿用现有约定）。

## 验收（必须真机、真 key）

1. 用户在设置里填热词（含 Fable 5、Sesame、casual talk），录一段故意说这些词的音频；
2. 转写稿里这些词应正确出现；结果页 meta 行「复盘用时（转写 X 秒·极速）」确认仍走极速版
   （若带 corpus 后掉到慢速通道 = 极速版不支持该参数，触发上面的去 corpus 重试逻辑有 bug）；
3. 清空热词再录一段，确认无回归。

## ⚠️ 施工三铁律（本项目已栽过的坑）

- 改 `mobile/js/core/*.js` 后必须 `cd santu_app/mobile && npm run build`（重建
  core.bundle.js）**再** `bash packaging/build.sh`，否则新逻辑不进冻结包且自检查不出。
- 打包前 `python3 -m py_compile santu_app/server.py` + `node --check`。
- 只改本规格点名的四处；活到你为止，不转包。做完更新 HANDOFF.md + CHANGELOG.md。
