## 背景 / 已定事实（不要重新决策）
- **误触方案 A 已上线**：书签放置改由**左**页边书签槽（`#bkSlot`/`#bkSlotA`）。所以本批**圆点贴一律走右页边**（mockup 的半圆 `border-radius:28px 0 0 28px` 平边贴右缘、弧朝版心），与左侧书签槽天然分家。
- **划选工具条已存在**：`#selbar`（`showSelectBar`/`showPhraseBar`/`placeBar`/`hideBar`），mouseup 选区→浮出 `讲解·追问` + `🗑️`。`.phrase` = 现有「词块讲解」高亮（marker 赭金 + 讲 badge）。`_selRange/_selText/_selSent` 存待处理选区。`wrapPhrase(range,text,sent)` 用 surroundContents（失败退 extract/insert）把选区包成 span——**本批新高亮复用同一套包裹技术**。
- **持久化既有事实**（决定两模式做法不同，务必遵守）：
  - **文章模式**：`doSave()` 存整块 `innerHTML` 快照（`{html}`→后端 `self._snapshot`）。所以文章模式的内联高亮 span **随 HTML 快照天然持久，零后端改动**（和 `.phrase`/`.bookmark` 一样）。
  - **书模式**：`save_session({page,chapter,bookmarks})` 只存 records，章节每次从 `article_html` 重渲。所以书模式高亮/圆点**必须用锚点 records**，新增 `highlights`/`dots` 两个字段，**完全照抄 `bookmarks` 字段的 plumbing**，并在章节渲染后按锚点重新应用。

## Phase 1：划选荧光笔高亮 + 整段色卡 + 划选工具条（②③④⑥）

### 1. CSS（照 mockup ②④）
```css
.reading .hl{border-radius:4px;padding:.02em .1em;-webkit-box-decoration-break:clone;box-decoration-break:clone;cursor:pointer}
.reading .hl.para{display:block;border-radius:10px;padding:13px 16px;margin:0 -16px .9em}   /* 整段色卡：更淡 .16 洗色由内联 style 给 */
```
- 高亮 span 背景走内联 `style="background:var(--wX)"`（X=色 key）。整段色卡另用 `.hl.para` + 更淡洗色（洗色值×约0.6，或直接给 `rgba(...,.16)`；照 mockup ④ 用 `.16`）。
- **叠加层级（③）**：高亮是内联 span，`.w`/`mark.vocab`（赭金 .30）/`.sent` 在其**内部**嵌套 → 天然在高亮之上；书签发丝线是 `#bkLayer` 覆盖层从中穿过。**不要**给 `.hl` 设 z-index 抢层。验证点：高亮内的生词仍可点、赭金底仍在上层。

### 2. 划选工具条改造（⑥）—— 扩 `showSelectBar`
选区 mouseup 后的 `#selbar` 从只有「讲解·追问 🗑️」扩成：`[讲解·追问] | ●●●●●●●(七色) | [复制]`。
- `showSelectBar` 的 innerHTML 改为：`讲解·追问` 按钮（`.sb-go`，行为不变）+ 分隔 + 7 个 `.sb-c` 色点（`data-c="A".."P"`，背景 `var(--cX)`）+ 分隔 + `复制` 按钮（`.sb-copy`）。样式照 mockup ⑥ `.selbar .c`（15px 圆点）。
- `#selbar` click 分发新增：
  - `.sb-c` → `applyHighlight(colorKey)`：用 `_selRange` 复用 `liftRangeToWords`+包裹逻辑（**新函数 `wrapHighlight(range,colorKey)`**，克隆 `wrapPhrase` 但 class=`hl`、无 explanation、带 `data-hl` 元数据），清选区、`hideBar()`；书模式还要 push 锚点 record + `scheduleSave()`；文章模式 `scheduleSave()`（HTML 快照）。
  - `.sb-copy` → 复制选区纯文本到剪贴板（优先后端 `copy_text` pbcopy，见 app.py 已有 `copy_text`；前端调用 `api().copy_text({text:_selText})`），`hideBar()`。
- **讲解必须留在第一位、行为完全不变**（`explainSelection` 一字不改）。划选取词/词块讲解现有逻辑不许被破坏——回归验证：选一段→点讲解仍走 `.phrase`+LLM。

### 3. 点已有高亮 → 小工具条改色/移除
点 `.hl` span（类比 `reopenPhrase`）：`#selbar` 进入 `mode='hl'`，显示 `七色(改色) | 移除`。移除=解开 span（`unwrapHl`，克隆 `removePhrase`）。改色=改内联背景 + 更新 record。别和「点 `.phrase` 重开讲解」冲突（`.hl` 无讲解）。

### 4. 书模式锚点持久化（关键）
- 数据：`let _highlights=[]`（镜像后端 `highlights` 字段，随 renderBook `r.highlights` 装载，形如 `_bookmarks`）。每条：`{id, ch, blockOrd, start, end, color, kind:'span'|'para'}`。`start/end`=在 `blocks[blockOrd]`（=`_bkBlocks()[blockOrd]`）**textContent 里的字符偏移**（遍历该块所有后代 text 节点按序累加，含 `.sent`/`.w` 内的文字）。`kind:'para'`=整段（start=0,end=len）。
- **落锚**（applyHighlight 时，书模式）：从包好的 `.hl` span 反推它在所属 block 的字符区间——包裹前先算 `range` 起止相对 block 的字符偏移（walk block 的 text 节点累加到 range.startContainer/startOffset）。存 record。
- **重应用**（每次 `renderChapterContent`/`gotoChapter` 渲完、`_initPager` 之后）：新函数 `_applyHighlightsForChapter()`——对当前 `bookIdx` 的每条 record，取 `blocks[blockOrd]`，把字符区间 `[start,end)` 映射回一个 Range（字符偏移→text 节点+offset），用 `wrapHighlight` 包成 `.hl`（surroundContents 失败退 extract/insert）。**在生词高亮已经渲染进 DOM 之后**执行，让 `.w`/`mark.vocab` 嵌进 `.hl` 内部。整段色卡直接给 block 加 `.hl.para` class + 背景（不用包 Range）。
- app.py：`save_session` 书分支 + `load_archive` 书分支 各加 `highlights` 透传，**照抄 `bookmarks` 那两处**（`args.get("highlights")`、`rec.get("highlights") or []`、renderBook 返回体加 `highlights`）。renderBook 前端接 `r.highlights`。

### Phase 1 验收（preview_eval 量数字 + 截图）
1. 书模式：选一段→点色点→出 `.hl` span、背景=对应洗色、record 入 `_highlights`；**换章再回来该高亮仍在**（重应用生效）；存档恢复（注入 highlights 重渲）后仍在。
2. 生词叠加：高亮内的 `mark.vocab` 仍可点讲解、赭金底在高亮之上（getComputedStyle 量层级/可点）。
3. 讲解不回归：选区点「讲解·追问」仍生成 `.phrase` + 走 LLM（canned）。
4. 整段色卡：整段模式给 block `.hl.para`，向两侧微出血（margin:0 -16px）。
5. 复制：`.sb-copy` 调 `copy_text`（canned stub 返回 ok 即可）。
6. 文章模式：同样能划选上色、点高亮改色/移除；`doSave` 后 innerHTML 快照含 `.hl`。
7. 截图过眼：七色在暖纸上和谐、不刺眼；藕粉与陶红/黛紫同族。

## Phase 2：圆点贴（右页缘半圆）+ 页码条同色刻度（⑤）
> Phase 1 验收通过后再做。圆点贴的**锚定复用书签那套**（`_bkBoundaries`/`_bkFragPage`/`_bkFragRect`/`_bkBoundaryAtY`）。

- 数据：`let _dots=[]`，每条 `{id, ch, ord, side, color, label}`（label 可空，≤3 字，默认色=上次用色，首次用藕粉 `P`）。**书模式**：新 `dots` 字段照抄 `bookmarks` plumbing（save_session/load_archive/renderBook 三处 + 前端 `_dots=r.dots||[]`）。**文章模式**：与书签一致——dot 记为 `.reading-inner` 内 absolute 元素随 HTML 快照持久（复用 `blockAtY` 锚到块）。
- **放置手势（主会话已定，别自创）**：**右**页边槽 `#dotSlot`(书,pager-viewport 内 absolute right:0 width:42) / `#dotSlotA`(文章,fixed 贴列右缘，`_bkPositionSlotA` 的镜像 `_dotPositionSlotA`)。与左侧书签槽**完全对称**：默认一道极淡右页边竖线，hover/armed → 加深 + ＋跟随行高（＋用"上次用色"着色）+ 右缘浮出半圆落点预览。**点一下=直接在最近段落边界落一枚半圆**（用 `_bkBoundaryAtY` 求边界；单击即放，不先弹色板，保持和左槽一样干脆）。开卷一次性提示复用 `_bkSlotIntro` 同款（可传 id）。
  - **改色/加标签/删除=点已落的半圆**→浮出一个 mini popover（复用 selbar 那种浮条样式）：`七色圆点(改色) | [2–3字标签输入] | 移除`。标签存进 record，hover 半圆时以 `.dotlabel` 浮出（在半圆左侧、朝版心）。**放置一步到位、编辑靠点圆点**——不要在放置时就逼用户选色/填字。
  - 与左侧书签槽/落线**互不干扰**：书签在左、圆点在右，各自 slot 各自覆盖层。
- 渲染（书模式）：覆盖层（类比 `_renderBkLayer`）在**右**页缘画半圆 `.halfdot`（14×28，`border-radius:28px 0 0 28px`，`right:0`，背景 `var(--cX)`），Y=锚点 fragment 行高、只画当前页当前章。hover 浮 `.dotlabel`（在半圆左侧、朝版心）。正反两页同高度都显示（book 是单页视口，无需，此条针对双页无关，忽略）。
- 页码条刻度：`.pager-bar` 内加同色小圆点 `.tick`，left=该 dot 在**全书**的位置分数（`ord`/章估算 or 章序占比），点击直达那页/章。全书索引。
- 颜色/样式逐像素照 mockup ⑤（`.dotwrap/.halfdot/.dotlabel/.pagerbar .tick`）。

### Phase 2 验收
- 右页缘半圆落点=书签那套锚定（换页/换章/字号变正确重画、只画当前页）；标签 hover 浮出；页码条刻度同色、点击跳转；与左书签槽/落线互不干扰（同页放一书签一圆点，各在各边）。
