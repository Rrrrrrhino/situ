## 0. 你要做什么（一句话）

在主窗 index.html 做「收获集」界面：入口→册页（双排序/整区变色·墨化/揭晓显影/对照）→
导出自包含 HTML →勾选喂复述。

## 1. 入口（time-to-content：一步直达）

- 文章页 `#artTools` 胶囊与书页 `#barTools` 胶囊，「仅标记」按钮**右侧**各加一枚
  「收获集」按钮（id：`artSheafBtn`/`bkSheafBtn`；图标用麦穗或册页线稿，stroke-width 1.7 同族）。
- 点开 = 右侧 `.panel` 区新增一个 tab 页（与 讲解/生词本 并列第三 tab「收获」）——**不要**
  另起全屏 overlay：右栏即对照阅读（正文在左，收获集在右，天然并列，用户点名要对照）。
- tab 徽标带条数：`收获 12`。无标记时空状态一句话：「开着「仅标记」读，划到的都会在这里
  长成讲解。」（书页语气，不卖萌不催促）。

## 2. 条目渲染（照抄样机③页「甲改」组的 CSS，数值不复议）

- 每条目：词块标题（衬线 600、原档色）+ 档签（淡底印刷章：档色字+档色 13–15% 衬底，
  **绝不要白字实心块**）+ `snapped:true` 时一行小注「你划的是「raw」—— 已归位到它所在的完整词块」
  + 左列例句 3–4 句竖排（虚线分隔，首句带「原文 · 第 N 段/章」srcmark 小注）+ 右列释义/用法注。
- **整区变色·墨化**：标题=原档色（一档 `#5f7a58`/二档 `#b0894a`/三档 `#8d8271`）；例句+释义+
  用法注=档色墨（`#4f6849`/`#8a683a`/`#6b6357`）。例句内核心词块**仅加粗 700，color:inherit，
  无任何高亮底**。srcmark 保持 muted。
  为什么两阶：原档色铺正文时二档发飘、三档像禁用文字（样机③页可复看）。
- 右栏宽度就是现成 `.panel` 372px——样机是双列布局，进 372px 右栏时改为**上下结构**：
  标题行→例句列→释义（释义不再单独右列，缩进 12px 细线左框）。其余 token 不变。
- 「原文·第 N 段」的 N：blockOrd 是含标题的原始块序，显示时**文章模式减 1**（跳过 h1），
  书模式显示「第 ch+1 章」。点 srcmark → 正文滚到该块并 800ms 高亮一下（复用 .current 呼吸）。

## 3. 排序与揭晓

- 顶部 seg 切换「按章节序 ⇄ 按档位」（样机②页样式）；默认按章节序。
- **揭晓=显影，无门槛**：tab 打开内容即刻可读（骨架仅用于 status 还在 pending 的条目，
  `.ink-in` 渐次补齐——轮询 `get_sheaf` 3s 直到 pending 空）；档色以 `transition:color .55s`
  洇入标题、档签同步淡入、条目间 90ms 错峰。**绝不要**：全屏动画、翻牌、需点击的「开启」步骤。
  每个 tab 打开只显影一次（session 内再开不重播）。

## 4. 导出自包含 HTML

- 册页顶部「导出」小按钮（ghost）→ 生成完全自包含 HTML（内联全部 CSS，双列版式=样机③页
  甲改组原版；无外链、无 JS 依赖），经桥 `export_sheaf_html({doc_id})` 落到 `OUTPUT_DIR`，
  弹系统 Finder 显示（学现有导出的做法）。文件名：`收获集-{标题截20字}-{YYMMDD}.html`。

## 5. 复述衔接（γ，并入本批）

- 每条目 hover 出小勾选框；选中 ≥1 条时册页底部浮出一行「用这 N 条去复述 ›」→ 调
  `open_review_retell`，把选中 chunks 作为目标表达传给复盘窗（批⁶ 已有 retell 管线，
  `localapi.js retell_targets` 优先吃传入 chunks——只需把 chunks 塞进 pending 条子的载荷，
  接口对齐 `server.set_retell_pending` 现状，别改复盘窗内部逻辑）。
- 未选任何条目时**不显示**这行（不催促）。

## 6. 范围围栏

- 动：`index.html`（收获 tab/入口按钮/册页渲染/导出触发）、`app.py`（`export_sheaf_html`、
  retell 载荷塞 chunks）。
- **不碰**：核批的划痕层逻辑、批α 的生成管线内部、`mobile/`、selbar/七色高亮、生词本 tab。
- 不删任何现有代码；正文渲染管线只加不改。

1. preview（源码起静态服务+stub 桥，参考核批做法）：无标记空态文案✓；8 条 mock sheaf 渲染
   双排序切换✓；显影只播一次✓；pending 条目骨架→ink-in✓。截图过眼：③页甲改配色成套、
   372px 右栏无溢出错位。

4. `node --check`（抽出内联 script）+ `py_compile` 过；`bash packaging/build.sh` 装机，
   grep 坐实新 id 已 baked。
