## 0. 交付什么
- 从**错题本列表**进"挑几条练"(多选 → 写作训练)；**复盘结果底栏的「练这几条」**也路由到这里
  (带上那次复盘的 errors/naturalness 条目)。
- 训练屏：顶部列出要练的几条(原话 → 改法，当提示)；一个 `textarea` 让用户用这些表达写一段英文；
  「提交批改」。
- 批改结果屏：**逐条**显示"用上了吗 / 用对了吗 / 你文中的原句 / 一句反馈" + 一条整体总评；
  可一键把练对的条目「标记掌握」。被练到的条目 `reviewCount++`、`lastSeen` 更新。
- 复用 4.1/错题本的视觉语言(暖纸 `#faf8f4`、深蓝 `#16335c`、archival 高亮、霞鹜文楷讲解)。
  **不新造框架、不新造配色。**

## 1. 批改 Prompt（逐字照搬，判断核心，一字不改）
端口进 `js/core/llm.js`，命名 `WRITING_CHECK_SYSTEM` / `WRITING_CHECK_TEMPLATE`。

### WRITING_CHECK_SYSTEM
```
你是一位资深英语写作老师，正在帮"中文母语"的学习者做针对性的造句 / 写作训练。学习者刚复习了几个表达（来自他的错题本），现在用它们写了一段英文。你要逐条检查他有没有真的用上、有没有用对，给出鼓励但具体的反馈。

你只能输出一个 JSON 对象（不要任何额外文字，不要 markdown 代码块）：
{
  "items": [
    {
      "target": "要练的表达，原样回显",
      "used": true 或 false,
      "correct": true 或 false 或 null,
      "quote": "他文中实际用到该表达的片段，逐字照抄；没用到则空字符串",
      "feedback": "一句中文：用对了就具体夸一句；用错了说清怎么改；没用上就提示怎么用"
    }
  ],
  "overall": "一到两句中文总评：整体是否自然、最值得表扬的一点、最该改进的一点"
}

要求：
- used：宽松匹配——时态、单复数、词形变化都算"用上了"。
- correct：仅判断"这个目标表达本身用得对不对"；used=false 时 correct 给 null。
- 只盯这几个目标表达，别去揪文中其它无关小错（那是复盘的事，不在这里做）。
- quote 必须逐字照抄学习者原文片段，便于定位。
- 鼓励但具体，所有反馈用简短中文。
```

### WRITING_CHECK_TEMPLATE（user 消息）
```
【要练的表达】
{items_block}

【学习者写的英文】
{text}

请逐条检查并只输出那个 JSON 对象。
```
- `{items_block}` 每行一条：`- {original} → {correction}`（correction 即错题本里的改法/更自然说法）。

## 2. llm.js：新增 `checkWriting(args)`
- 签名 `checkWriting({ items, text }) -> { ok, items:[{target,used,correct,quote,feedback}], overall, model, error? }`。
  入参 `items` 为 `[{original, correction}]`。
- **结构镜像 `reviewSpeech`**（4.1 已建）：读 store 配置/JSON mode/_extractJson 容错/**空答重试**
  (v4-pro `max_tokens` 给足 ≥1536)。校验：`items` 强制成数组，缺字段的条目丢弃不整体失败。
- 缺 key 返回 `{ok:false, error:'未配置 API Key，去设置里填'}`。

## 3. store.js：训练记录（轻量）
- 复用现有 `mistakes` 表：训练后对练到的条目 `reviewCount = (reviewCount||0)+1`、`lastSeen=now`；
  用户点"标记掌握"才置 `mastered=true`（**不自动判掌握**，保自主感）。
- 新增 `trainings` 表(可选，做训练历史；keyPath=id)：
  `{ id, createdAt, mode:"writing", itemIds:string[], text, result:{items,overall}, model }`。
  bump DB_VERSION(2→3) + onupgradeneeded 建表，照现有模式。

## 4. localapi.js：新增方法
```ts
make_writing_drill({ ids?, count? }) ->
  { ok, items:{ id, original, correction, type }[], prompt }
//   ids 给定就用这些错题本条目；否则自动挑 count(默认3)条未掌握的(unmastered, 按 addedAt 或随机)。
//   prompt = 客户端可直接拼的提示串，如"用下面这几个表达，自然地写 3–5 句话（或一小段）："
//   （出题不需要 LLM，本地拼即可，省一次调用）
check_writing({ itemIds, text }) ->
  { ok, items:{ target, used, correct, quote, feedback }[], overall, model, error? }
//   内部：按 itemIds 取 mistakes 条目 → llm.checkWriting → 写 trainings 一条
//   → 对 used&&correct 的条目 reviewCount++/lastSeen 更新 → 回结果
//   （不自动标 mastered；标掌握走现有 set_mistake_mastered）
```

## 5. UI（复用现有视图/列表/sheet 框架）
- **入口**：① 错题本列表加一个「挑几条练」按钮 → 进多选态(条目前出选择框) → 选完点「练这 N 条」
  → 训练屏。② 4.1 复盘结果底栏的「练这几条」(目前占位) → 改成带本次复盘条目进训练屏。
- **训练屏**：
  - 顶部"要练的几条"提示卡：每条 `原话 → 改法`（小卡，archival 色，跟错题本一致）。
  - 一段写作提示(make_writing_drill.prompt)。
  - `textarea`(占位"用上面的表达写一段英文…") + 「提交批改」按钮(深蓝)。
- **批改结果屏**：
  - 逐条卡片：左侧状态标 —— 用对=橄榄绿✓、用错=赭橙✗、没用上=灰‹—›；显示 `target`、你的原句
    `quote`(有则高亮该片段)、`feedback`(中文)。
  - 底部 `overall` 总评(暖纸卡)。
  - 每条练对的旁边给「标记掌握」(走 set_mistake_mastered)；练完可「再练一组 / 回错题本」。
- 文案中文为主、英文衬线、讲解楷体；状态色与 4.1 一致(别新造)。
