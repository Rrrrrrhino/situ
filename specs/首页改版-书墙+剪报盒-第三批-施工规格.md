# 四土 · 第三批 首页改版（书墙 + 剪报盒 + 单一智能入口）· 施工规格

> 判断轻产出重：设计已定稿（用户逐版拍板），本规格照做即可。**品味/取舍拿不准就停下问主会话，别自作主张改设计。活干到你这一步为止，不要再转派 builder。**

## 目标
把四土首页从「所有条目混排成书脊」改成 **方向 B**：上方 **书架（书墙）**、下方 **剪报盒**，顶部一个 **单一智能入口**。书籍与文章分区呈现；**两区一屏全见、不用滚动**（≥700px 高窗口）。

## 视觉源（唯一真源 · 逐像素照它）
`specs/_harness/homepage-mock.html`（切到「方向 B · 上下分区」那版，就是默认版）。里面的 `.dropzone / .book-wall / .hero / .rest-grid / .clip-band / .clip-scroll / .clip / .wall-head` 全套 CSS + `heroHTML/bookHTML/clipHTML/coverHTML/wallHTML` 全套渲染逻辑，**直接移植进 `santu_app/index.html`**，颜色用项目已有 `:root` 变量（mockup 里的 `--ink #3a5570` 用项目里的 `--navy #4a6b85` 或新增一个深蓝油墨变量，别引亮蓝）。

关键版式（已定稿，别改）：
- **书墙**：`display:flex;gap:20px`。第一本 = **最近在读的那本书**（list_library 里 mode==book 的第一条），做成 **hero**（封面 150px 宽、放大），紧挨着其余小封面（`.rest-grid`，minmax(86px)，书多了自动折两行）。hero 就是一枚更大的普通封面 + 作者，**不要**加「最近在读」文字标、**不要**压暗渐变（用户明确否掉了），靠尺寸和头位区分即可。
- **剪报盒**：`.clip-band`（顶部一道 `border-top` + 「剪报盒」head），文章做成 `.clip-scroll` 横向卡片（`.clip` 236px，左缘一道套色、来源·标题·两行摘要·日期）。
- **单一入口 `.dropzone`**：一个框，文案「把书或文章丢进这里 / 拖入 EPUB 电子书，或粘贴文章链接 / 文本 —— 自动识别，各归各墙」。
- 竖向留白按 mockup 的 `.dir-b` 收紧值，保证一屏（实测书墙 225px、剪报盒 662px 收尾）。

## 数据（已确认）
- `api().list_library()` → 数组，newest-first，每条含 `mode`('book'|'article')、`title`、`source`、`saved_at`、`current_page`、`chapter_count`、`epub_path`(书才有)、`id`。
- **书墙** = `items.filter(i=>i.mode==='book')`；hero = 该列表第 0 条；其余进 `.rest-grid`。
- **剪报盒** = `items.filter(i=>i.mode==='article')`，做成 `.clip` 卡（source=`i.source`、title=`i.title`、日期=从 `saved_at` 格式化、摘要暂用 `i.source` 或留空占位；无正文摘要就先省略那两行）。
- 进度：书用 `current_page`/`chapter_count` 估一个百分比或「第 X 章」；估不出就不显示。

## 改造点（`santu_app/index.html`，外科手术，别动无关逻辑）
1. **首页 DOM**：`#hint.welcome` 里 `.wel-top`（问候 + `#drop` + sites）+ `#shelfWrap>#shelf` 这块，改成：`[可保留一行紧凑问候] + .dropzone + 书墙容器 + 剪报盒容器`。保留一句小问候即可（复用 `setWelGreeting`），别撑破一屏。
2. **渲染**：新写 `renderHome()`（替代/包住 `renderShelf`）——拉 `list_library`、按 mode 分流、填书墙(hero+rest)与剪报盒。空状态各自兜底（无书/无文章各给一句淡提示 + 末位「＋添加」）。
3. **单一入口接线**（**复用现成逻辑，不要重造**）：
   - `.dropzone` 点击 → `fileInput.click()`（现有 `#file` 选文件）。
   - 拖放：全窗口 drag-drop 已在（`window 'drop'` @~3102，`.epub/.txt` 自动路由）；把 `.dropzone` 也作为高亮目标即可，落盘仍走现有 handler。
   - 粘贴链接/文本：`.dropzone` 里可放一个隐藏/浮出的输入，或点击时聚焦现有 `urlInput`；Enter → `run(value)`（`run()` 已自动识别 URL/文本/文件后缀，见 2781 + 3102）。**不新增识别逻辑**。
   - 书墙「＋添加书籍」「剪报盒 ＋添加文章」都指向同一入口动作。
4. **打开/删除照旧**：点 hero/小封面/剪报卡 → `openArchive(id)`；删除 → 复用 `.spine-del`/`deleteFromShelf` 那套（改成 hover 出 ✕ 调 `delete_archive`）。`goHome()`/`initWelcome()` 落到这个新首页（它们调 `renderShelf`——把它们改调 `renderHome`）。radial 菜单、阅读态不受影响。
5. **封面**：本批只做 **字排兜底封面**（暖纸 + 顶部套色带 + 衬线书名 + 作者，套色按 index/hash 轮转 mockup 那 7 色）。**真封面抽取＝第二批，本批不做**（见文末）。

## 验收标准（贴证据，别贴结论）
- preview 截图：首页＝单一入口 + 书墙(hero 大 + 小封面紧挨) + 剪报盒，**一屏全见**（量 `#剪报盒容器` bottom ≤ 窗口高，给出数字）。
- hero = 最近那本书、放大；其余小封面；文章只进剪报盒、不混入书墙（给出分流后各数量）。
