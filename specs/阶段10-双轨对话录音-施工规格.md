# 四土 · 阶段 10：双轨对话录音 + 对话复盘 + 偷学（施工规格）

> 用户拍板：放弃浏览器扩展抓取路线，直接做 Mac 双轨录音。原理：**麦克风轨 = 我，系统声音轨 = AI**
> ——说话人分离由物理通道天然解决，对 App 无差别（Sesame 网页、ChatGPT 桌面版、任何出声的东西都能录）。
> **用户追加硬要求（2026-07-06）：外放（不戴耳机）也必须分清两轨**——麦轨走系统级回声消除（§1 AEC）
> + 转写后串音过滤（§3）双保险；任何交付物里不得出现「建议戴耳机」作为主方案。
> 闭环：菜单栏一键录 → 停止后落盘两轨 WAV → 四土复盘页看到待复盘对话 → 服务端转写（带
> utterance 时间戳）→ 交织成对话稿 → 我的话走现有复盘管线（含词块盯防）→ **偷学**（从对方的话里
> 挑值得学走的表达，一键进词块库）。
>
> 主会话已设计：架构、STEAL prompt（§5，逐字照搬一字不改）、数据流、UI。**活到你为止，不许转包。**
> 汇报贴证据不贴结论。

## 1. Swift 菜单栏录音器「四土对话录」（新 SPM 包 `对话录/`）

- **交互**：菜单栏一个细线麦克风图标。菜单项：「开始录音」/「停止录音」（录音中图标变红点态、
  菜单显示已录时长）/「打开四土复盘」（`open http://127.0.0.1:18760`）/「退出」。无主窗口。
- **录音**：SCStream 音频双输出（照 kuailu 模式；video 尺寸给最小值、视频帧直接丢弃）。两轨各自
  实时转 16kHz 单声道 16-bit PCM 写 WAV（流式写，别攒内存；WAV header 收尾时补长度）。
  安全上限 2 小时自动停。
- **落盘**：`~/Documents/situ/data/dualtrack/<yyyyMMdd-HHmmss>/` 下 `me.wav`（麦）、`ai.wav`（系统）、
  `meta.json`（`{startedAt, durationSec}`）。停止时目录写完整后再落一个 `ready` 空文件（避免半成品被读）。
- **权限**：系统声音需「屏幕录制」权限、麦轨需「麦克风」权限。首次点开始若无权限，弹一段
  NSAlert 指引（去系统设置哪里开、开完重启 App）。打包必须 ad-hoc 签名（照 kuailu build-app.sh），
  否则 TCC 不记忆授权。
- **回声消除（AEC）——本批红线，用户明确要求：外放（不戴耳机）也必须分清两轨**。
  麦轨**不用** SCStream 的 `captureMicrophone`，改用独立 `AVAudioEngine` + Apple 系统级回声消除
  （VoiceProcessing I/O，FaceTime 同款：系统从麦信号里减掉正在外放的声音）：
  ```swift
  let engine = AVAudioEngine()
  try engine.inputNode.setVoiceProcessingEnabled(true)   // 必须在 engine.start() 之前
  if #available(macOS 14.0, *) {
      // 绝不压低 AI 外放音量（VPIO 默认会 duck 其它音频，这里显式关掉）
      engine.inputNode.voiceProcessingOtherAudioDuckingConfiguration =
          AVAudioVoiceProcessingOtherAudioDuckingConfiguration(
              enableAdvancedDucking: false, duckingLevel: .min)
  }
  let tapFmt = engine.inputNode.outputFormat(forBus: 0)  // VPIO 下采样率可能是 24k/48k，别硬编码
  engine.inputNode.installTap(onBus: 0, bufferSize: 4096, format: tapFmt) { buf, _ in
      // AVAudioPCMBuffer → 16kHz mono Int16 → meWriter.append（AudioTrackConverter 加一个
      // AVAudioPCMBuffer 入口，内部 AVAudioConverter；现有 CMSampleBuffer 入口留给系统轨）
  }
  try engine.start()
  ```
  - SCStream 侧同步改：`captureMicrophone = false`、去掉 `.microphone` output，SCStream 只负责系统声音轨。
  - **失败回落**：`setVoiceProcessingEnabled` / `engine.start()` 任一 throw → Log 记原因，回落到
    SCStream `captureMicrophone` 老路（老代码保留作 fallback 路径），`meta.json` 写 `"aec": false`；
    成功则写 `"aec": true`。
  - 菜单文案：**不出现任何「建议戴耳机」字样**。录音中菜单小字显示「回声消除已开启」；
    回落时显示「回声消除不可用，外放会串音（建议耳机）」——只有回落态才提耳机。
  - 已知取舍（注释里写给未来维护者）：VPIO 把麦输入变成语音频段（电话音质），转写场景无损；
    双讲（你和 AI 同时说话）时 AEC 可能压低用户音量，属正常；AEC 消不干净的残留由 §3 串音过滤兜底。
- **交付**：构建脚本 `对话录/build.sh`（swift build release → 手搓 .app bundle，做法照 kuailu
  build-app.sh / situ tools/build_app.sh）→ 产出拷到 `~/Desktop/四土对话录.app`。

## 2. server.py：三个新端点

- `GET /api/dualtrack_list` → 扫 `data/dualtrack/`，返回**有 `ready` 且无 `done` 标记**的会话：
  `[{dir, startedAt, durationSec}]` 按时间倒序。
- `POST /api/dualtrack_transcribe`：body `{dir, appid, token}`。服务端读该目录 `me.wav`/`ai.wav`，
  各自按 **10 分钟**切片（PCM 数据层直接切，重造 WAV header），逐片走现有火山代理逻辑
  （submit/query，协议照 `HANDOFF-手机原生-④语音训练.md` §★），**request 里加 `"show_utterances": true`**，
  解析 `result.utterances[].{text, start_time, end_time}`（毫秒），每片时间戳加上片偏移后合并，
  返回 `{me:[{text,start,end}], ai:[{text,start,end}]}`。**防御回落**：若响应没有 utterances 字段，
  该轨整体降级为 `[{text: result.text, start:0, end:durationMs}]`（仍可用，只是无法细粒度交织），
  并在返回里带 `warnings`。凭证只经手不落盘不打日志。
- `POST /api/dualtrack_done`：body `{dir}` → 目录里落 `done` 标记文件（不删音频，用户可回溯）。
- 路径安全：`dir` 参数只允许 `data/dualtrack/` 下的直接子目录名（拒绝 `..`/绝对路径）。

## 3. 交织 + 对话复盘（js/core/localapi.js + review.js）

- localapi 新增：
  ```ts
  list_dualtrack() -> [{dir, startedAt, durationSec}]
  process_dualtrack({ dir }) -> { ok, reviewId, ...review_speech v2 返回全量, dialog, steals, error? }
  ```
- `process_dualtrack` 内部流程：
  1. 调 `/api/dualtrack_transcribe`（凭证读 IndexedDB volc_*）；
  2. **串音过滤（第二道防线，交织之前跑，无条件启用）**：对 me 轨每条 utterance，
     同时满足以下两条才判为串音、丢弃：
     - 与 ai 轨 utterances 的时间重叠时长 / 该条自身时长 ≥ 0.5；
     - 归一化文本（小写、去标点、压空白）后 token 数 ≥ 3，且 me 条 token 对「重叠的 ai 文本」的
       containment（me∩ai 交集 token 数 / me token 数）≥ 0.6。
     宁放过不错杀：短插话（"yeah exactly"）因 token<3 或相似度低必须存活。丢弃条数计入返回
     `warnings`（如「已过滤 2 条疑似串音」）。抽成独立纯函数 `filterBleed(me, ai)` 以便单测。
  3. **交织**：两轨 utterances 按 `start` 归并排序，speaker 标 `me`/`ai`；相邻同 speaker 合并成一个
     turn → `dialog: [{speaker, text}]`；两轨都空 → 返回错误「没转出内容」；
  4. 我的话拼接 = `text`；对话全文（`我: …` / `对方: …` 逐行）= `context`（超 20000 词截断+warning）；
     调现有 `review_speech({text, context, source:'dual'})`（词块盯防自动生效）；
  5. 调 `llm.stealFromDialog`（§5）得 `steals`；
  6. `dialog` 和 `steals` 一并存进该条 review 的 `result`（store reviews 行加这两个可选字段，不 bump
     schema 也行，向后兼容即可）；调 `/api/dualtrack_done`。
- **UI（review.js）**：
  - 输入屏顶部：`list_dualtrack` 非空时显示一张暖纸卡「🎙 有 N 段对话录音待复盘」（细线图标别用
    emoji），点开列表（时间 + 时长），点某段 → 进度态（「转写中…约 X 分钟音频，耐心等」）→
    `process_dualtrack` → 直接进结果屏。失败 toast 真实原因，会话保留在待复盘列表可重试。
  - **结果屏两处新增**（仅 source='dual' 的复盘）：
    1. 原文卡位置换成**对话稿卡**：逐 turn 渲染，speaker 标签「我」（深蓝小签）/「对方」（暖灰小签），
       我的 turn 里照旧做错误高亮；默认折叠露前几个 turn。
    2. 「**偷学**」区块（放在「用得好」之后）：橄榄绿系标题；每条卡片：`expression`（衬线，显眼）+
       `quote`（小字引用）+ `why`（中文）+ `example`（衬线）+「存为词块」按钮
       （`add_chunk({text:expression, meaning:why, example, source:'steal', sourceRef:reviewId})`，
       已存显示「已在词块库」）；区块头一个「全部入库」。
  - 历史屏：source='dual' 的条目加对话小图标；重开时对话稿/偷学区正常重渲染。

## 4. llm.js：`stealFromDialog`

- 签名 `stealFromDialog({ dialog }) -> { ok, steals:[{expression, quote, why, example}], model, error? }`。
  `dialog` = `[{speaker:'me'|'ai', text}]`，拼成模板里的 `{dialog}`（`我: …`/`对方: …` 逐行）。
- 用 `_loadReviewConfig`（精批档）；maxTokens 1536（空答重试 3072）；temp 0.3；`_extractJson` 容错；
  校验：steals 强制数组、缺 expression/why 的条目丢弃、最多留 4 条。
- 对方 turns 全为空（比如只录到自己）→ 不调 LLM，直接返回 `{ok:true, steals:[]}`。

## 5. STEAL Prompt（逐字照搬，一字不改）

### STEAL_SYSTEM

```
你是一位资深的英语口语外教。下面是"中文母语"学习者和一位母语级对话者（AI 或真人）的对话转写。你的任务是替学习者从【对方】的话里"偷学"：挑出最值得他学走、马上能用的地道表达。

你只能输出一个 JSON 对象（不要任何额外文字，不要 markdown 代码块）：
{
  "steals": [
    {
      "expression": "值得偷学的表达（词块/搭配/句式），从对方原话提取，还原成原形（如动词用原形）",
      "quote": "对方用到它的那句原话，逐字照抄",
      "why": "一句中文：它好在哪、什么场合用",
      "example": "一条贴近日常生活的新英文例句，示范怎么用"
    }
  ]
}

挑选标准：
- 只挑 2-4 条，宁缺毋滥。优先：①学习者自己没用过、但按他本轮表现出的水平"跳一跳够得着"的 ②口语高频、真的常用的 ③比学习者已有说法明显更地道的。
- 不挑：生僻词、书面腔、单个普通单词、学习者本轮已经自己用对的表达、以及转写噪音（不完整的句子碎片）。
- 从学习者本轮的话可以看出他的水平：挑对他有增量的，不挑他明显早就会的。
- 对方的话太少或没有值得偷的，就少给或给空数组，绝不硬凑。
- 所有中文简短、具体。
```

### STEAL_TEMPLATE

```
【对话转写（"我"=学习者，"对方"=母语级对话者）】
{dialog}

请挑出值得学习者偷学的表达，只输出那个 JSON 对象。
```

## 7. 红线

- 录音数据是个人数据：`data/dualtrack/` 不进 git（加 .gitignore）。
