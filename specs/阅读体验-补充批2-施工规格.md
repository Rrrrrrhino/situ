## #1 读物精选浮层：滚动时不再关闭，改为「跟随来源卡」保持 steady

**现状**（index.html 末，约 L3987-3989）：
```
window.addEventListener('scroll', ()=>{ if(_discOpenId) closeDiscList(); }, true);
window.addEventListener('resize', ()=>{ if(_discOpenId) closeDiscList(); });
```
popover 是 `position:fixed`，一滚就关——用户觉得不 steady。

**改法**：滚动时**重新定位**（跟着来源卡走），而不是关闭：
- scroll 处理器：`if(_discOpenId){ 用 rAF 节流，取当前打开来源卡 srcEl，调 positionDiscList(list, srcEl) 重定位。}` 这样 popover 始终贴着卡片、像页面的一部分，滚动顺滑。
- **仅当来源卡本身滚出视口**（`rect.bottom<0 || rect.top>innerHeight`）时才 `closeDiscList()`——卡都看不见了，popover 自然该收。
- resize 仍 `closeDiscList()`（resize 布局剧变，重定位不划算，直接关更干净）。
- 用 `requestAnimationFrame` 合并高频 scroll 事件，避免抖动/性能问题。

**验收**：preview 打开 popover → 程序化滚动少许 → popover 仍显示且重新贴合来源卡（量 `list.bottom` 相对 `srcEl.top` 仍是 8px gap）；把来源卡滚出视口 → popover 关闭；resize → 关闭；`console_logs` 零报错、滚动无明显抖动。

## #3 读物精选：The Conversation 可便捷切 global / us / uk（feed 子标签）

**现状**：上一批为提速把 TC 砍成只拉 global 单 feed（`DISC_OUTLETS` 里 TC 只剩 `{name:'全球',...global...}`）。用户想能在 global/us/uk 间便捷切换。

**改法**（用「按需加载的 feed 子标签」——既给选择又不牺牲速度，比原来"并行拉 3 个合并"更快）：
1. `DISC_OUTLETS` 恢复 TC 三个 feed：`{name:'全球',url:'.../global/articles.atom'}`、`{name:'美国',url:'.../us/...'}`、`{name:'英国',url:'.../uk/...'}`。CNA 保持单 feed。
2. **popover 头部**（`renderHd`，约 L3938）：若 `outlet.feeds.length>1`，在头部渲染一排 feed 子标签按钮（全球/美国/英国），当前选中高亮（复用 `.dlh-act`/`.disc-list-hd` 语言，新增一个 `.dlh-tab`/`.dlh-tab.on` 轻样式）。单 feed 的来源（CNA）不显示子标签。
3. **状态**：记当前打开来源的 `feedIdx`（默认 0＝全球）。切 feed 子标签 → 重新 `discLoadOutlet(outlet, feedIdx)` 渲染该 feed 文章。
4. **缓存键改为 `outlet.id + ':' + feedIdx`**（`_discCache`）——每个 feed 各自缓存，切回不重拉。`↻刷新` 只刷当前 feed。
5. **预取**（`discPrefetchAll`）：每个来源**只预取 feedIdx 0**（全球），保持预取轻量；其余 feed 点到才拉。
6. `discLoadOutlet` / `discOpenOutlet` 签名加 `feedIdx` 参数（默认 0），沿途传递。切 feed 后 popover 高度变 → 末尾再 `positionDiscList` 一次。

**验收**：preview 点 TC → 头部出现「全球|美国|英国」子标签、默认全球有文章；点「英国」→ `preview` 侧计数器确认发了 uk feed 请求、列表换成 uk 文章；再点回「全球」→ 无新请求（缓存命中）。CNA 无子标签、行为不变。首页预取只预取各来源全球 feed（计数器验证）。

## #4 讲解区着色：文章模式也上色 + 扩到「单词本身/本义/这里」三处（书本文章一致）

**现状根因**（已定位）：
- 书本回看走 `_load_chapter_internal` 重新分析 → `self._last` 带 classifier → 点词有 `daily_rank`→`freq_band` → 上色。
- **文章回看**（`load_archive` 文章分支，app.py L1718）只还原 `article_html` 快照、`self._last=None` → 点词 `classifier=None`→`daily_rank=None`→`_freq_band(None)=None` → **无 band、不上色、无角标**。
- 前端 `explanationHtml` 当前只给**标题词** span 上色（L1551 `fv=FREQ_VAR[r.freq_band]`），本义/这里未上色。

**改法**：
### 4a. 后端：文章回看重建轻量 classifier
`load_archive` 文章分支把 `self._last = None`（L1718）改为构造一个只需频率表的 classifier（`classify_word` 只依赖频率表+spaCy，不需要整篇分析报告，已核实）：
```python
try:
    classifier = VocabClassifier(user_level=self._level or "cet4-6")
except Exception:
    classifier = None
self._last = {"title": rec["title"], "source": rec["source"],
              "classifier": classifier, "vocab_order": [], "report": None}
```
（`VocabClassifier` 顶部已 import。`_cur_meta` L1446 读 `self._last["title"/"source"]` 已满足；无代码读 `self._last["report"]`，None 安全。）这样文章回看点词也能拿到 `freq_band`，且短语/句「重点词汇」上色（#A2）在回看文章里也生效。

### 4b. 前端：本义 / 这里 也按频率色
`explanationHtml`（L1550-1551）给「本义」「这里」的 `.val` 加频率色（fv 已算好）：
```
${r.literal?`<div class="ex-row"><span class="lbl">本义</span><div class="val"${fv?` style="color:var(${fv})"`:''}>${escapeHtml(r.literal)}</div></div>`:''}
${r.contextual?`<div class="ex-row"><span class="lbl">这里</span><div class="val"${fv?` style="color:var(${fv})"`:''}>${escapeHtml(r.contextual)}</div></div>`:''}
```
标题词已上色（保持）。三处同一 `var(--fX)`，书本/文章一致（同一函数）。无 band 时回落默认色。

**验收**：Python 侧——构造一个 article 存档，`load_archive` 后 `self._last["classifier"]` 非 None、`explain_word` 对 'vacate' 返回 `freq_band='C'`（重定档后按新阈值，见 #5——vacate 14813→B）。前端——preview 注入带 `freq_band` 的词讲解，`getComputedStyle` 量「标题词/本义 val/这里 val」三处 color 都＝对应 `--fX`；无 band 时三处都回落默认色。截图过眼：中文本义/这里染频率色后仍清晰可读、暗色和谐（低饱和档案盘）。

## #5 频率档重定档：A≤4000 / B≤21000 / C≤33000 / D≤41000 / E>41000

**现状**：`_freq_band`（app.py L107-122）阈值 3000/8000/15000/30000。SUBTLEX 口语语料压低了 overweight(16383)/nutritious(20686) 等书面常用词 → 落 D，用户要它们进 B。

**改法**：
1. `_freq_band` 阈值改为（label/name 不变）：
```
if rank <= 4000:  A 最常用
if rank <= 21000: B 常用
if rank <= 33000: C 较常用
if rank <= 41000: D 进阶
else:             E 生僻
```
更新函数上方注释里的旧数值说明（15k/30k 那段）以免误导。
2. **已收藏词条的旧 band 刷新**：已存的 notebook/global 条目里 `freq_band` 是旧阈值算的（overweight 存着 'D'），不会自动变。**在读取/下发时按 `daily_rank` 重算 band**，让阈值调整对存量词也即时生效：
   - `get_notebook`（下发生词本列表处）与全局生词本序列化处：对每条有 `daily_rank` 的词，`b=_freq_band(daily_rank)`，覆盖 `freq_band=b["label"]`/`freq_name=b["name"]`。无 `daily_rank` 的（legacy）保持原样。
   - 这样用户打开生词本 / 讲解区回看，overweight/nutritious 立即显示 B，无需重新点。
3. **不动** `COMMON_RANK_CUTOFF`（vocab.py，控制正文高亮的 common/rare 与是否标注，属另一套，用户没点名）。

**验收**：Python——`_freq_band(16383)=='B'`、`_freq_band(20686)=='B'`、`_freq_band(14813)=='B'`、`_freq_band(22135)=='C'`、`_freq_band(2152)=='A'`、`_freq_band(42859)=='E'`（逐个断言）；构造一个存着 `daily_rank=16383, freq_band='D'` 的旧词条，`get_notebook` 下发后该条 `freq_band=='B'`（重算生效）。前端生词本/讲解区对应变色一致。
