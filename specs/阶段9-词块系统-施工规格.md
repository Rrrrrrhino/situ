## 1. 数据模型（store.js，bump DB version）

新表 `chunks`：
```ts
{ id, text:string,              // 词块本体，如 "be inclined to"
  meaning:string,               // 一句中文义/用法（可空串）
  example:string,               // 一条典型例句（可空串）
  source:"reading"|"review"|"steal"|"manual",
  sourceRef:string,             // bookId / reviewId 等，可空串
  addedAt:number, lastDrilled:number,
  drillCount:number,            // 被练总次数（drill+日常复盘命中都算）
  correctRefs:string[],         // 用对的场次 id 列表（drillId 或 reviewId，去重）
  correctTopics:string[],       // 用对时的话题（仅展示用）
  mastered:boolean, star:boolean }
```
**掌握规则（铁律）**：`correctRefs.length >= 3` 时自动 `mastered=true`（三个不同场次用对）。
同一场次同一词块只计一次。helper 命名照现有 store 风格（chunksPut/chunksList/chunksGet/chunksDelete）。

## 2. Prompt（逐字照搬，一字不改）

### 2a. 新增 CHUNK_DRILL_SYSTEM（llm.js）

```
你是一位资深的英语口语外教，正在帮"中文母语"的学习者做词块（chunk）刻意练习。学习者刚拿到几个目标词块，现场说或写了一段英文来使用它们（口语转写可能带不完整、重复、口头语，不要苛责）。你要逐个词块裁决他用得对不对——他最需要的是确定性的反馈：到底哪里对、哪里不对、正确的用法长什么样。

你只能输出一个 JSON 对象（不要任何额外文字，不要 markdown 代码块）：
{
  "topic": "用 4-8 个中文字概括他这段在说什么",
  "items": [
    {
      "chunk": "目标词块，原样回显",
      "used": true 或 false,
      "quote": "他文中实际使用该词块的片段，逐字照抄；没用到则空字符串",
      "verdict": "correct | unnatural | collocation | grammar | context 五选一；used=false 时为 null",
      "comment": "两句以内中文：correct 时确认对在哪里（形式/搭配/语境都要点到）；其余档说清错在哪、正确该怎么说；没用到就提示一个自然的用法方向",
      "examples": ["该词块的典型正确例句；verdict 不是 correct 时必须给 2-3 条；correct 时给 0-1 条拓展用法"]
    }
  ],
  "extraErrors": [
    { "original": "目标词块之外的明显错误片段，逐字照抄原文", "correction": "改正后的说法", "why": "一句中文" }
  ],
  "overall": "一两句中文总评"
}

verdict 判定标准：
- correct = 这个词块本身用得完全对：形式对（时态/单复数/词形）、搭配对、语境合适，三者都对才给。
- unnatural = 没有语法错，但母语者不会在这里这么用（语域不合，或搭配生硬但能懂）。
- collocation = 词块内部或前后搭配用错（如介词错、动宾搭配错）。
- grammar = 词块相关的语法错（时态/主谓一致/词形）。
- context = 词块形式没错，但用错了场合、词不达意。
- used 宽松匹配：时态、单复数、词形变化都算"用上了"。
- 裁决要果断：对就是对、错就是错，不许含糊其辞——学习者要的是确定性。
- examples 必须地道、常用、贴近口语，别造书面腔。
- extraErrors 只放目标词块之外"明显"的错（真语法错/用词错），最多 3 条；轻微问题不放，主战场是那几个词块。
- 所有中文简短、具体、不客套。
```

### 2b. 新增 CHUNK_DRILL_TEMPLATE

```
【目标词块】
{chunks_block}

【学习者的英文】
{text}
{context_block}
请逐个词块裁决，只输出那个 JSON 对象。
```
- `{chunks_block}`：每行 `- {text}` + 有 meaning 时接 `（{meaning}）`。
- `{context_block}`：有练习话题时 `\n\n【本次练习话题】\n{topic}`，否则空串。

### 2c. 新增 CHUNK_TOPIC_SYSTEM / TEMPLATE（出话题，小调用）

```
给定几个英语词块，出一个即兴口语话题，让"中文母语"的学习者能自然地一次用上所有这些词块。话题要贴近日常生活或观点表达，别出成作文题。只输出一个 JSON 对象（不要任何额外文字）：
{ "topic_zh": "一两句中文话题描述", "opener_en": "一句英文开场提示，帮他起头" }
```
```
【词块】
{chunks_block}

请出题，只输出那个 JSON 对象。
```

### 2d. 阶段8 复盘 prompt 的逐字插入（让日常漫谈盯词块）

**只做以下三处插入，其余一个字不动**（llm.js 里的 REVIEW_V2_SINGLE_SYSTEM 与 REVIEW_V2_EDIT_SYSTEM）：

**插入①**（两个 SYSTEM 都做）：在 JSON 结构中 `"minor": [ 与 priority 同结构的对象 ]` 这一行之后、
`}` 之前，插入一行逗号并接：
```
  "chunkFeedback": [
    {
      "chunk": "他正在练的词块，原样回显",
      "quote": "他文中使用（或明显试图使用）该词块的片段，逐字照抄",
      "verdict": "correct | unnatural | collocation | grammar | context 五选一",
      "comment": "一两句中文：对就确认对在哪，错就说清怎么改"
    }
  ]
```

**插入②**（两个 SYSTEM 都做）：在判定/整理要求列表的最后追加一条：
```
- 【进行中的词块】给出时：检查他有没有用到（或明显试图用）其中的词块，用到的逐条给 chunkFeedback（verdict 标准：correct=形式搭配语境全对；unnatural=没语法错但母语者不这么用；collocation=搭配错；grammar=语法错；context=用错场合）。没用到的词块不要出现在 chunkFeedback 里，绝不硬凑；没给词块清单时 chunkFeedback 给空数组。词块相关的问题只进 chunkFeedback，不要重复放进 priority / minor。
```

**插入③**（两个 TEMPLATE 都做）：在 `{mistakes_block}` 之后另起一行加 `{chunks_block}`。
`{chunks_block}` 构造：有进行中词块时
`\n\n【进行中的词块（他正在刻意练习，留意是否用到）】\n- chunk1\n- chunk2 ...`，否则空串。

## 3. llm.js

- `chunkDrill({ chunks:[{text,meaning}], text, topic }) -> { ok, topic, items, extraErrors, overall, model }`：
  用 `_loadReviewConfig`（精批档）；`_extractJson` + 空答重试照旧；maxTokens 3072（重试 6144）、temp 0.3。
  校验：items 里 chunk 不在传入清单的丢弃；verdict 不在五档集合→置 "unnatural" 兜底；examples 强制数组。
- `chunkTopic({ chunks }) -> { ok, topic_zh, opener_en }`：主配置即可（便宜档），maxTokens 512。
- `reviewSpeechV2` 加可选参数 `chunks:[{id,text}]`，按 §2d 拼 `{chunks_block}`；返回值加
  `chunkFeedback`（校验：chunk 不在传入清单的丢弃、verdict 非法丢弃）。

## 4. localapi.js

```ts
add_chunk({ text, meaning?, example?, source?, sourceRef? }) -> {ok, id}   // text 去首尾空白；同 text 已存在则返回旧 id
list_chunks({ view?, q? }) -> Chunk[]        // view: unmastered(默认)|star|mastered|all；q 搜 text/meaning
delete_chunk({ id }) / set_chunk_star({ id, star? }) -> {ok}
suggest_chunk_topic({ ids }) -> {ok, topic_zh, opener_en}
check_chunk_drill({ ids, text, topic? }) ->
  { ok, drillId, topic, items:[{chunk, chunkId, used, quote, verdict, comment, examples,
    progress:{correct:number, need:3}, justMastered:boolean}], extraErrors, overall, model }
```
`check_chunk_drill` 内部：调 `chunkDrill` → 逐条更新 chunks：`drillCount+=1`、`lastDrilled=now`；
verdict==="correct" 时把 `drillId` 加进 `correctRefs`（去重）、topic 加进 `correctTopics`；
`correctRefs.length>=3` → `mastered=true` 且该条 `justMastered:true`（UI 庆祝一下——**完成才给的
bonus，不做事前诱饵**）。

`review_speech` 升级：调用前取进行中词块（未掌握，star 优先、再按 lastDrilled 最久优先，cap 10）传给
`reviewSpeechV2`；返回的 chunkFeedback 同样更新对应 chunk 计数（场次 id 用 reviewId，同一复盘同一
词块只计一次），结果随 review 一起落库（result 里加 chunkFeedback 数组），返回值带 progress/justMastered。

## 5. UI（review.js + app.js + style.css）

沿用暖纸/深蓝/赭橙/暖金/橄榄绿，**不新增色**。verdict 胶囊配色：correct=橄榄绿、unnatural=暖金、
collocation/grammar/context=赭橙。五档中文标签：完全正确 / 不够自然 / 搭配错 / 语法错 / 语境不合。

1. **词块库屏**：复盘头部加「词块」按钮（细线 SVG，与错题本/历史并列；`_onBack` 补路由）。
   列表照错题本模式：tab（未掌握/重点/已掌握/全部）+ 搜索 + 条目（英文 text 衬线、meaning 小字、
   进度点 `●●○ 2/3`、star、来源小 chip：读/盘/手/偷）。展开：example + 加重点/删除。
   顶部两个动作：「＋添加」（小 sheet：text 必填 + meaning/example 选填）、「练一组」。
2. **选词块练习**：复用错题本的多选模式挑 2-5 个 →（可选）点「给我个话题」调 `suggest_chunk_topic`
   显示 topic_zh + opener_en → 大 textarea + 复用现有录音按钮（转写追加）→「提交裁决」。
   「练一组」不选时自动配：未掌握中 star 优先、lastDrilled 最久优先取 4 个。
3. **裁决结果屏**：overall 顶卡 → 逐词块卡片：verdict 胶囊 + quote（引用原句）+ comment +
   examples 列表（衬线）+ 进度点（justMastered 时胶囊换成「🎉 已掌握」用橄榄绿、给一个轻微的
   完成动画，别浮夸）→ extraErrors 小节（赭橙，最多 3 条）→ 底栏「再练一组 / 回词块库」。
4. **复盘结果屏**：chunkFeedback 非空时，在「重点区」之后插入「词块反馈」区块，卡片样式同裁决结果屏
   （带进度点/justMastered）。
5. **复盘结果 & 错题本条目**：每张 error/naturalness 卡加「存为词块」小按钮（text=correction/better，
   meaning=why 截断，source="review"）。已存则显示「已在词块库」。
6. **阅读讲解面板**（app.js）：讲解动作区加「存为词块」按钮——存当前查询词/短语（meaning 取讲解里
   现成的中文义第一句，example 取讲解例句第一条；没有就空串），source="reading"，sourceRef=当前
   book/article id。只加按钮和一次 `api('add_chunk',...)` 调用，**不改讲解逻辑**。
