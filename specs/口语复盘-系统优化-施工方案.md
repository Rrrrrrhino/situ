# 口语复盘子界面 系统优化 施工方案
> 2026-07-07 由 fable5 会话定盘，交 Opus 会话执行。
> **动手前必读**：`~/Downloads/apply/style-codex/风格圣经.md`（验收标准，不是参考资料）＋ 本文件全文。
> 范围：`santu_app/mobile/`（review.js / app.js 录音卡部分 / index.html 复盘浮层头部 / style.css）。
> **数据层一律不动**：IndexedDB 结构、LocalApi、server.py API、错题/词块/历史的存储格式，全部保持原样。

## 0. 起点（fable5 窗已落地，以现状为准；2026-07-07 两批）
- 复盘独立窗（REVIEW_ONLY）输入屏顶部已有**双轨录音卡**（标题「录音语料」，`review.js _loadRecCard`，复用 app.js 首页卡的全局函数 `_fillRecCard/_recStart/_recStop/_startRecPoll/_recPermGuide`）。
- 独立窗不再渲染单麦「录音说一段」按钮、也不再有「背景/话题」输入；文本区语义=「转写后的文本」。**单麦 getUserMedia 路保留**给手机版复盘输入屏和词块练习作答，不许删。
- **录音→复盘一条龙已落地（用户拍板的形态，别改回子屏流）**：每段录完的录音在输入屏顶部就地成 chip（时间+时长+✕丢弃，`_loadDualtrackCard` 的 REVIEW_ONLY 分支）；底部主按钮变「转写并复盘」，一键在本屏转写（转写稿落进文本区，`_doReviewFromRecording`）并直接出复盘结果。dualList 子屏只剩手机版在用。
- 另有 `_startRecWatch`（1.5s 轮询 recorder_status）：录音开始/停止翻转时整屏刷新输入屏，停止后 chip 立即冒出来。
- 录音失败**绝不静默降级成单麦**：失败明说＋权限指引（指引文案在 app.js `_recPermGuide`，授权对象是「四土对话录」）。这是铁律，重构时不许弄丢。
- **双轨引擎麦轨已根修**（Recorder.swift tap 显式单声道——装虚拟声卡的机器上 VPIO 聚合出 7ch，AVAudioConverter 7→1 降混会静默产全零；另有 micPeak 看门狗写 meta、server 转写端 meSilent 检测+跳过+精准报错）。重构 UI 时**别碰** 对话录/ 与 server.py 的这些逻辑。
- **晚批追加（同样别碰）**：SCStream 自动重启续录+补零对齐（跨窗口/退全屏不再断录）；权限=稳定证书「KuaiLu Codesign」签名+直接 exec 二进制（responsible=四土，单一授权主体，跨更新存活）；语料 chip 可点选（`RS.selectedDualDir`，选中态=墨框+提示，批β 只做视觉）；转写两轨+切片并行。

## 1. 现状清单（重构对象）
头部（所有屏共享）：‹返回 | 标题 | 历史⏱ | 词块⧉ | 错题本📋 | ⚙（REVIEW_ONLY 动态插入）。
每个 `_show*Screen` 手工 add/remove 三个头部按钮的 hidden——十几处调用点，忽隐忽现且互不一致。

| # | 屏 | 内容 |
|---|----|------|
| 1 | 输入屏 input | 双轨录音卡＋待复盘卡＋textarea＋背景话题＋复盘按钮（手机另有单麦按钮） |
| 2 | dualList | 待复盘对话列表＋转写进度态 |
| 3 | result | v2：meta/总评/对话稿(或原文卡)/重点区/词块反馈/用得好/偷学/完整清单/sticky底栏；另有 v1 降级渲染 |
| 4 | mistakes 错题本 | 4-tab seg(未掌握/重点/已掌握/全部)＋类型chips＋搜索＋挑几条练(多选) |
| 5 | drill 写作训练 | targets卡＋textarea＋提交批改 |
| 6 | check 批改结果 | 结果＋再练一次/回错题本 |
| 7 | history 复盘历史 | 搜索＋按日分组列表(mic/paste/dual 来源图标)→打开旧复盘 |
| 8 | chunks 词块 | 4-tab seg(同4)＋搜索＋「＋添加/练一组/挑几个练」 |
| 9 | chunkPick 练词块 | targets卡＋「给我个话题」＋textarea＋录音说一段(麦)＋提交裁决 |
| 10 | chunkResult 裁决结果 | 同 6 的同构体 |

核心病灶：**④⑧同构双份、⑤⑥与⑨⑩同构双份、头部四图标不平级、录完到结果要手动走三步、桌面窗是手机布局直接拉宽（无版心、大圆角满宽药丸按钮）**。

## 2. 信息架构：四图标 → 「三域一设置」
```
复盘（根屏）        积累（库）           练习（引擎，无常驻入口）
录 / 贴 二择        错题 | 词块 二 tab    从积累或结果屏带条目进入
待复盘队列          （共享同一套UI代码）
最近复盘3条＋全部›
```
- **头部收敛为两枚图标**：积累（书夹图标）＋ ⚙。历史入口移进根屏正文「最近复盘」区；练习只能从积累/结果屏带条目进入，不给常驻入口。
- **头部改「屏声明制」**：删掉所有手工 hidden 切换，每个屏渲染时声明 `{ title, headerButtons: [...] }`，一个函数集中应用。这是消灭忽隐忽现的根治法。
- 返回键逻辑（REVIEW_ONLY 藏根屏返回、子屏复显）沿用现有 MutationObserver 机制，行为不变。

## 3. 功能合并（能合则合，能简则简）
### 3.1 积累合体（④＋⑧ → 一屏）
- 新「积累」屏顶部一个 seg：「错题 | 词块」。切换只换数据源与条目渲染器，**4-tab 视图、搜索、多选、star/掌握/删除操作栏共用同一份实现**。
- 错题的类型 chips（语法/词不达意/搭配/不自然）只在错题 tab 显示；词块的「＋添加/练一组」只在词块 tab 显示。
- 数据 API 照旧分别调 `list_mistakes/list_chunks` 等，不动后端。

### 3.2 练习引擎（⑤⑥＋⑨⑩ → 一套）
- `_showPracticeScreen({ kind: 'mistake'|'chunk'|'mixed', items })`：targets 卡（kind 决定行渲染：原文→纠正 / 词块＋义）＋话题出题按钮＋textarea＋录音短答（麦克风路，通用化现有 `_bindChunkRecBtn`）＋提交。
- 提交按 kind 走 `check_writing` 或 `chunk_drill`（后端不动）；结果屏用同一个渲染器。
- 「给我个话题」现在只有词块有（`suggest_chunk_topic`）；错题练先不做出题，按钮只在 chunk kind 显示——**不加后端**。
- 顺手解锁混合练：积累屏多选可跨 tab 带条目（错题＋词块同场）→ kind='mixed'，提交时分别调两个 API 合并展示。若工作量超预期，mixed 可砍，前两步不许砍。

### 3.3 录音→复盘一条龙 ✅ 已由 fable5 落地（见 §0），本方案不再动它
- 定型形态（用户拍板）：录完就地成 chip → 点「转写并复盘」在本屏转写（稿落文本区）→ 直接出结果。失败 chip 留队列可重试，报错走 localapi 精准文案。
- 批β 只做它的**视觉**印刷化（chip/按钮），交互流程一字不改。

### 3.4 根屏（输入屏）重排
- 上：双轨录音卡「录音语料」（录音中=红点＋等宽计时，常驻可见）＋待转写 chips（现状保留）。
- 中：「转写后的文本」textarea＋「转写并复盘/复盘」主按钮（现状保留；独立窗无背景/话题）。
- 下：**「最近复盘」3 条**（日期戳＋主题＋来源小图标，点开即旧结果）＋「全部 ›」进完整历史屏。历史屏本体（搜索＋按日分组）保留，只是入口搬家。

## 4. 视觉印刷品化（对照风格圣经逐条）
- **版心**：独立窗内容区 max-width 680–720px 居中；两侧是纸的留白。手机窄屏不受影响。
- **细线版框替代卡片**：录音卡/待复盘卡/重点区块/targets 卡 → ≤1px 暖灰细线框＋无阴影＋圆角≤6px。
- **按钮印章化**：现在的满宽大圆角深蓝条（开始录音/复盘/提交批改）全部收窄——像一方印章或版框里的一行铅字：居中或右置、圆角≤6px、深蓝油墨底或深蓝描边字。**深蓝只做油墨不做涂料**：任何区块不许刷满屏深蓝底。
- **seg/chips 档案标签化**：压暗低饱和（琥珀/赭橙/橄榄/浅天蓝按圣经 §2 成套），当前项像贴上去的档案标签，不做发光高亮。
- **结果屏排版**：总评=题眉小字＋一段正文；对话稿默认折叠（现有）；重点区恒定 3–5 条（既有铁律）；「完整清单」默认折叠为一行小字注。sticky 底栏保留但印刷化（细线上框＋铅字按钮）。
- **录音中状态**＝印章红点＋等宽数字计时，绝不做呼吸发光/脉冲动画；动效只做纸的物理（滑入/叠上），150–300ms。
- 空状态文案照圣经 §8：一句话陈述，不道歉不卖萌（现有「暂无待复盘的对话录音」合格，照此风格补齐其他）。
- **交付前必须过圣经 §9 全部自查**，视觉改动施工方自己截图过眼（亮/暗 × 桌面宽/手机窄，合并断点一次截完）。

## 6. 铁律与风险（施工方必读）
1. **先 git commit 再动手**（安全绳）；review.js 已 2293 行，批α 拆分成 review-input / review-result / review-library / review-practice 数个平级 script（index.html 按序引入，无构建步骤），拆前单独 commit。
2. **手机版零回归**：所有桌面差异只由 `window.REVIEW_ONLY` 与「recorder_status 是否可达」门控；单麦路（手机复盘＋练习作答）不许删；v1 旧数据降级渲染（`_showResultScreenV1`）不许动。
3. **浮层 z 序**：设置浮层压序 bug 刚修过（`_ovTopZ` 递增），重构头部/浮层时不许回归——验「浮层打开」要看 z 序盖没盖住，不能只看 display。
4. 反强迫温柔：不加 streak/百分比/催促；练习与复盘的完成反馈=事后陈述（「N 条已存进错题本」这类），不做事前胡萝卜。
5. 打包后必须 grep 装机产物确认新符号已 baked（冻结包旧代码坑）。
6. 外科手术：与本方案无关的读书/词汇/精选模块一行不碰。

## 7. 验收清单（用户视角）
1. 复盘窗打开即见：录音卡（或录音中计时）、textarea、最近复盘——三样一屏内。
2. 头部只有 ‹、标题、积累、⚙，任何子屏进出不再忽隐忽现。
3. 录对话：开始→停止→chip 冒出→点「转写并复盘」直达结果，中途零子屏导航（已落地，改版后不许退化）。
4. 错题与词块在同一「积累」屏内切换，操作（搜索/筛选/多选/练）手感一致。
5. 从错题、词块、复盘结果三处发起练习，进的是同一个练习屏，说/写皆可。
6. 亮/暗两模式截图过圣经 §9 自查（无发光、无满屏深蓝、颜色成套、按钮似印章）。
